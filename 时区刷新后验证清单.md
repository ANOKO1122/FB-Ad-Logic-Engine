## 🔍 验证步骤

### 1. 验证时区刷新结果

**SQL 查询：**
```sql
-- 应该返回 0 行（所有时区已一致）
SELECT 
  ds.account_id,
  am.timezone_name as account_timezone,
  ds.timezone_name as daily_stats_timezone,
  COUNT(*) as mismatch_count
FROM daily_stats ds
INNER JOIN account_mappings am ON ds.account_id = am.fb_account_id
WHERE am.is_active = 1
  AND ds.timezone_name != am.timezone_name
GROUP BY ds.account_id, am.timezone_name, ds.timezone_name;
```

**预期结果：** 0 行（所有时区已一致）

---

### 2. 验证 today 查询只返回 today 数据

**测试方法：**
- 创建一个规则，条件为 `today spend > 0`
- 某广告今天没有花费，但过去 7 天有花费
- 规则应该返回 `false`（不匹配）

**SQL 验证：**
```sql
-- 检查 ad_snapshots 中是否有 last_7d 聚合数据混入
SELECT 
  account_id,
  ad_id,
  synced_at,
  spend,
  timezone_name,
  DATE(synced_at) as sync_date
FROM ad_snapshots
WHERE account_id = 'act_927139705822379'
  AND DATE(synced_at) = CURDATE()
ORDER BY synced_at DESC
LIMIT 10;
```

**预期结果：**
- 所有记录的 `spend` 应该是"今天"的真实值
- 不应该包含"过去7天合计"的聚合值

---

### 3. 验证滑动窗口同步正常

**观察日志：**
- 等待下一次定时任务执行（每 30 分钟）
- 或手动触发一次同步

**检查点：**
- Today 数据写入 ad_snapshots
- 按日数据更新 daily_stats
- 日志显示正确的统计信息

---

### 4. 验证冷数据归档正常

**SQL 查询：**
```sql
-- 检查最近归档的数据
SELECT 
  account_id,
  date,
  timezone_name,
  COUNT(*) as count,
  MAX(updated_at) as latest_update
FROM daily_stats
WHERE date >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY account_id, date, timezone_name
ORDER BY latest_update DESC
LIMIT 20;
```

**预期结果：**
- `timezone_name` 应该是账户实际时区（如 `Asia/Shanghai`）
- 每个账户的 `date` 应该是账户时区的自然日

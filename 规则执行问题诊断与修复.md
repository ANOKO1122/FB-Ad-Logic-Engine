# 规则执行问题诊断与修复

## 🔍 问题分析

### 现象
- ✅ 规则已创建（`id=138, enabled=1, is_simulation=1`）
- ✅ 36个账户都获取锁成功，开始执行规则
- ❌ **但没有看到后续的执行日志**（没有看到规则匹配、执行或跳过的日志）

### 可能的原因

根据代码分析，规则执行流程中有**3个静默跳过点**（没有日志输出）：

1. **用户不存在或未激活**（第194行）
   - 如果规则关联的 `user_id` 在 `users` 表中不存在
   - 或者用户状态不是 `'active'`
   - 会静默跳过，**没有日志输出**

2. **用户无权访问账户**（第217行）
   - 如果用户不是 admin，且 `owner_id` 不匹配
   - 会静默跳过，**没有日志输出**

3. **规则指定的 accountId 不匹配**（第226行）
   - 如果规则指定了 `account_id`，但与当前执行的账户不匹配
   - 会静默跳过，**没有日志输出**

## 🔧 诊断步骤

### 步骤 1：检查规则的用户ID

```sql
-- 查看规则的 user_id
SELECT id, rule_name, user_id, enabled, is_simulation 
FROM rules 
WHERE id = 138;
```

### 步骤 2：检查用户是否存在且激活

```sql
-- 替换 <user_id> 为步骤1查到的 user_id
SELECT id, role, owner_id, status 
FROM users 
WHERE id = <user_id> AND status = 'active';
```

**如果查询结果为空**：
- ❌ 用户不存在或未激活
- ✅ **解决方案**：激活用户或创建用户

### 步骤 3：检查用户是否有账户访问权限

```sql
-- 替换 <owner_id> 为步骤2查到的 owner_id
-- 替换 <account_id> 为任意一个账户ID（例如：act_4252903478277839）
SELECT fb_account_id 
FROM account_mappings 
WHERE owner_id = <owner_id> 
  AND is_active = 1 
  AND fb_account_id = '<account_id>';
```

**如果查询结果为空**：
- ❌ 用户无权访问此账户
- ✅ **解决方案**：在 `account_mappings` 表中添加账户映射

### 步骤 4：检查规则是否指定了 accountId

```sql
-- 查看规则是否指定了 account_id
SELECT id, rule_name, account_id, user_id 
FROM rules 
WHERE id = 138;
```

**如果 `account_id` 不为 NULL**：
- 规则只会在此账户上执行
- 如果当前执行的账户不匹配，会跳过

## 📊 常见问题与解决方案

### 问题 1：用户不存在或未激活

**症状**：
```
⚠️  [account_xxx] 用户 X 不存在或未激活，跳过 Y 条规则
```

**解决方案**：
```sql
-- 激活用户
UPDATE users SET status = 'active' WHERE id = <user_id>;

-- 或创建用户（如果需要）
INSERT INTO users (id, role, owner_id, status) 
VALUES (<user_id>, 'user', <owner_id>, 'active');
```

### 问题 2：用户无权访问账户

**症状**：
```
⚠️  [account_xxx] 用户 X 无权访问此账户，跳过 Y 条规则
```

**解决方案**：
```sql
-- 添加账户映射
INSERT INTO account_mappings (fb_account_id, owner_id, is_active) 
VALUES ('<account_id>', <owner_id>, 1);
```

### 问题 3：规则指定的 accountId 不匹配

**症状**：
```
⏭️  [account_xxx] 规则 "规则名称" (ID: X) 指定的账户 Y 不匹配，跳过
```

**解决方案**：
- **选项1**：将规则的 `account_id` 设置为 `NULL`（作用于所有账户）
- **选项2**：确保规则在正确的账户上执行

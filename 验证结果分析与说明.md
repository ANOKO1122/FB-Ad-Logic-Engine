## 日志解读：规则执行相关

### 1. "用户25无权访问此账户"

**含义**：
- 这是**权限检查**的正常行为
- 用户25（ID=25）在系统中存在，但**没有权限**访问当前正在同步的账户
- 系统会跳过该用户的所有规则，避免越权执行

**代码位置**：`server/services/cronService.js` 第234-237行

**逻辑说明**：
```javascript
// 检查用户是否有权限访问此账户
if (accountRows.length === 0) {
  console.log(`⚠️  [${lockedAccountId}] 用户 ${userId} 无权访问此账户，跳过 ${userRules.length} 条规则`)
  continue // 用户无权访问此账户
}
```

**这是正常的安全机制**，不是错误！

---

### 2. "规则指定的账户不匹配，跳过"

**含义**：
- 规则绑定了**特定账户**（例如 `act_4252903478277839`）
- 但当前正在同步的账户是**另一个账户**（例如 `act_927139705822379`）
- 系统会跳过该规则，避免在错误的账户上执行

**代码位置**：`server/services/cronService.js` 第247-250行

**逻辑说明**：
```javascript
// 如果规则指定了 accountId，必须匹配
if (ruleAccountId && ruleAccountId !== lockedAccountId) {
  console.log(`⏭️  [${lockedAccountId}] 规则 "${rule.ruleName}" (ID: ${rule.id}) 指定的账户 ${ruleAccountId} 不匹配，跳过`)
  continue
}
```

**这是正常的账户匹配机制**，确保规则只在正确的账户上执行！

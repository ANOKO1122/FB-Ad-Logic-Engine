## 🐛 解决的问题

### 问题 1：数据库会话时区问题
- **问题**：MySQL 命令行显示 `session_tz: SYSTEM`，但应用已设置为 UTC
- **解决**：确认 Node.js 应用的数据库连接正确设置为 UTC，MySQL 命令行是独立会话

### 问题 2：`getDataTimezone` SQL 错误
- **问题**：`DISTINCT` 和 `ORDER BY` 冲突
- **解决**：重构 SQL 查询，使用子查询避免冲突

### 问题 3：分页问题导致前端看不到所有广告
- **问题**：`getAdInsights` 没有处理分页，只返回第一页数据
- **解决**：实现分页逻辑，添加 `limit: 200` 优化

### 问题 4：`last_7_days` 和 `last_30_days` 天数计算错误
- **问题**：返回 8 天（期望 7 天）、31 天（期望 30 天）
- **解决**：调整 `days` 参数计算逻辑

### 问题 5：`today` 查询返回空数组
- **问题**：`adIds` 为 `null` 时构建了 `ad_id IN (null)` 的 SQL
- **解决**：修改 SQL 查询构建逻辑，允许查询所有广告

### 问题 6：`last_7_days` 返回空数组（时区不匹配）
- **问题**：`daily_stats` 使用 `Asia/Shanghai`，查询使用 `UTC`，时区过滤不匹配
- **解决**：实现合并查询 + 智能时区匹配

## ⏰ 时间归因延迟问题

### 问题描述

Facebook 广告数据存在归因延迟（Attribution Delay）：
- 用户点击广告后，可能需要几小时甚至几天才会完成转化
- 当天的数据可能不完整，需要等待后续更新

### 解决方案

**已实现：** `syncAccountSlidingWindow` 函数

**功能：**
- 同步 Today + Past 7 Days 数据
- 解决 Facebook 归因延迟导致的「迟到事件」
- 使用 ON DUPLICATE KEY UPDATE 更新过去一周数据

**代码位置：** `server/services/ingestorService.js` 第 179-299 行

### ⚠️ 问题：未在定时任务中调用

**当前状态：**
- ✅ 函数已实现
- ❌ 未在定时任务中调用
- ❌ 需要手动触发或通过 API 调用

**建议：**
1. **方案 1（推荐）**：在定时任务中添加滑动窗口同步
   - 频率：每 30 分钟或每小时执行一次
   - 只同步 Today 数据（Past 7 Days 数据通过 ON DUPLICATE KEY UPDATE 更新）

2. **方案 2**：保持现状

## 🔧 建议的改进

### 1. 添加滑动窗口同步到定时任务（可选）

**目的：** 解决时间归因延迟问题

**实现：**
```javascript
// 在 cronService.js 中添加
// 每 30 分钟执行一次滑动窗口同步（Today + Past 7 Days）
cron.schedule('*/30 * * * *', async () => {
  console.log('🔄 开始滑动窗口同步（解决归因延迟）...')
  try {
    const [accounts] = await pool.query(`
      SELECT fb_account_id as account_id, owner_id, COALESCE(timezone_name, 'UTC') as timezone_name
      FROM account_mappings 
      WHERE is_active = 1
    `)
    
    for (const account of accounts) {
      try {
        await syncAccountSlidingWindow(
          account.account_id,
          account.owner_id,
          account.timezone_name
        )
      } catch (error) {
        console.error(`❌ 账户 ${account.account_id} 滑动窗口同步失败:`, error.message)
      }
    }
  } catch (error) {
    console.error('❌ 滑动窗口同步失败:', error.message)
  }
})
```

**频率建议：**
- 每 30 分钟：平衡数据新鲜度和 API 调用频率
- 或每小时：减少 API 调用，但数据更新延迟更长

## 📝 验证步骤

### 验证定时任务是否正常运行

1. **查看日志：** 观察定时任务的执行日志
2. **检查数据：** 确认 `ad_snapshots` 表有最新数据
3. **检查归档：** 确认 `daily_stats` 表有归档数据

### 验证冷数据归档

```sql
-- 检查最近归档的数据
SELECT account_id, date, COUNT(*) as count, MAX(updated_at) as latest_update
FROM daily_stats
WHERE date >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY account_id, date
ORDER BY latest_update DESC
LIMIT 20;
```

### 验证时区更新

```sql
-- 检查账户时区配置
SELECT fb_account_id, timezone_name, updated_at
FROM account_mappings
WHERE is_active = 1
ORDER BY updated_at DESC;
```

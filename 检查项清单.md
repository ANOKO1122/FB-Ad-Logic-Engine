# 系统状态检查项清单

本清单列出「系统状态一键检查」脚本会检查的项目，以及需要登录后查看的项目。

---

## 一键脚本会检查的项（无需登录）

| # | 检查项 | 说明 | 未通过时常见原因 |
|---|--------|------|------------------|
| 1 | **后端 API 服务** | 进程是否在跑、端口是否可访问 | 未执行 `npm run dev:server` 或端口被占用 |
| 2 | **Facebook Token** | 是否已配置 | `.env` 中未配置 `FACEBOOK_ACCESS_TOKEN` |
| 3 | **MySQL 数据库** | 能否连接、库是否存在 | MySQL 未启动、主机/端口/账号密码错误、库名不存在 |
| 4 | **统一心跳锁** | 是否有残留锁导致任务无法执行 | 上次进程异常退出未释放锁；可执行 `node server/scripts/force-release-heartbeat-lock.js` 释放 |

---

## 需登录后查看的项（脚本不执行）

| # | 检查项 | 说明 | 查看方式 |
|---|--------|------|----------|
| 5 | **Cron 定时任务** | 是否在跑、上次执行时间与耗时 | 浏览器登录 → 「系统状态」页，或 `GET /api/system/health`（带 cookie） |
| 6 | **各账户同步状态** | 每个 FB 账户最近一次同步是否及时（healthy / stale / error） | 同上 |
| 7 | **熔断器状态** | Token 是否被限流锁定（`is_system_locked`） | 同上 |
| 8 | **失败任务队列** | 最近 1 小时失败任务数量（`queue.pending_count`） | 同上 |

---

## 如何运行一键检查

在项目根目录执行：

```bash
node server/scripts/check-all-status.js
```

运行前请确保：

- 若要看「后端 API + Token」为通过，需先启动后端：`npm run dev:server`
- 若要看「MySQL + 心跳锁」为通过，需先启动 MySQL 服务

---

## 相关脚本与接口

| 用途 | 命令或接口 |
|------|------------|
| 一键检查（本清单 1～4 项） | `node server/scripts/check-all-status.js` |
| 仅检查心跳锁 | `node server/scripts/check-lock-status.js` |
| 强制释放心跳锁 | `node server/scripts/force-release-heartbeat-lock.js` |
| 无需登录的存活检查 | `GET http://localhost:3001/api/health`（端口以 .env 为准） |
| 需登录的完整系统状态 | `GET http://localhost:3001/api/system/health`（需带登录 cookie） |

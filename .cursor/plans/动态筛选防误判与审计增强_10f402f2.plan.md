### 1.5 ID 归一化（act_ 修复）— 显式约定

| 目的 | 保证写入 rules.target_ids / target_by_account 及预览结果的 ID 从源头就是「单层 act_」格式，避免 act_act_ 等脏数据进入配置表；配合「取消回写」，配置表不再被污染。 |
|------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| 改前 | 方案未显式写出，实现上 rulesService 已有 syncTargetByAccount 使用 parseCompositeId 做归一化，但未在「动态筛选防误判」方案中与 act_ 修复、取消回写形成闭环说明。 |
| 改后 | 在方案中**明确约定**：① **保存规则时**：createRule/updateRule 必须通过现有 syncTargetByAccount（内部使用 [targetIdUtils.parseCompositeId](server/utils/targetIdUtils.js) / normalizeAccountId）对 target_ids 做归一化后再落库，禁止将未归一化的 act_act_ 写入 rules。② **执行/预览时**：凡生成复合 ID（act_xxx:id）写入或返回给前端的路径（如 [dynamicScopeService.previewDynamicScope](server/services/dynamicScopeService.js) 返回 object_ids），统一通过 normalizeCompositeId(accountId, objId) 生成，确保与 rules 侧格式一致且无重复前缀。 |
| 实现确认 | rulesService 已在 createRule/updateRule 前调用 syncTargetByAccount，已满足「保存时归一化」；dynamicScopeService 的 previewDynamicScope 已使用 normalizeCompositeId。方案层面需在 1.1 或 1.4 旁注明：保存与预览路径均需经 ID 归一化，并在 Code Review 时核对所有写 rules.target_ids 或返回复合 ID 的入口。 |

## 二、审计追溯能力（含计数维度）

### 2.1 历史表字段（在 v2 基础上补充）

- **表**：`rule_matched_objects_history`
- **既有/已有**：object_count、filter_snapshot、filter_hash（v2 已规定）。
- **新增计数维度**：**manual_count** INT、**dynamic_count** INT（可为 NULL 以兼容旧数据）。

| 字段 | 含义 | 用途 |
|------|------|------|
| object_count | 当次最终集合大小 \| (Dynamic ∪ Manual) − Exclude \| | 与现有一致 |
| dynamic_count | 当次动态筛选得到的 ad 数量（union 前） | 若某条历史 dynamic_count 从 5 突增到 100，可快速判断是筛选条件异常 |
| manual_count | 当次手动目标展开到 ad 的数量（union 前） | 若 manual_count 变化，可判断为用户勾选或 target_ids 变更导致 |

### 2.2 实现要点

- 在 [calculateMatchedAdIdsForRule](server/services/dynamicScopeService.js) 中，在计算 finalAdIds 后，已有 dynamicSet.size、manualSet.size（或等价长度）；在 refreshDynamicTargetsForRulesInAccount 写入 rule_matched_objects_history 时，除 object_count 外，传入 **dynamic_count**、**manual_count**（仅对 status === NORMAL 且有变动的记录写入）。
- 迁移：在 038 或新增迁移中为 rule_matched_objects_history 增加 `manual_count INT NULL`、`dynamic_count INT NULL`（若表已存在且无此二列）。

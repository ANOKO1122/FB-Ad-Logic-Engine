# FB-Ad-Logic-Engine 系统部署与运维说明

## 一、运行架构

| 组件 | 运行方式 | 端口 | 说明 |
|------|----------|------|------|
| **后端 API** | systemd 服务 `fb-ad-logic-engine` | 3001 | Node.js Express（`NODE_ENV=production`，内存上限 1.5G），含定时任务调度 |
| **前端静态资源** | Nginx Docker 容器 | 80 / 443 | 反代后端 API + 托管 dist/ 静态文件 |
| **MySQL 数据库** | 独立服务 | 3306 | 规则、快照、审计日志等数据存储 |

---

## 二、关键路径

| 路径 | 说明 |
|------|------|
| `/root/work/FB-Ad-Logic-Engine` | 项目根目录 |
| `/root/work/FB-Ad-Logic-Engine/dist` | 前端构建产物（bind mount 到 Nginx） |
| `/root/work/FB-Ad-Logic-Engine/deploy/nginx-fb-ad-docker.conf` | Nginx 配置文件 |
| `/root/work/FB-Ad-Logic-Engine/server/server.js` | 后端入口 |
| `/etc/systemd/system/fb-ad-logic-engine.service` | systemd 服务定义文件 |
| `/root/work/FB-Ad-Logic-Engine/logs` | 后端日志目录（Winston 按日轮转） |

---

## 三、重启命令

### 完整重启（推荐）

```bash
# 1. 重新构建前端
cd /root/work/FB-Ad-Logic-Engine && npm run build

# 2. 重启后端
systemctl restart fb-ad-logic-engine
```

### 仅重启后端（前端无改动时）

```bash
systemctl restart fb-ad-logic-engine
```

### 仅重启后端 + 语法预检（推荐，部署代码改动时）

```bash
cd /root/work/FB-Ad-Logic-Engine && node --check server/services/ingestorService.js && systemctl restart fb-ad-logic-engine
```

### 仅重新构建前端（后端无改动时）

```bash
cd /root/work/FB-Ad-Logic-Engine && npm run build
# dist/ 为 bind mount，构建完立即生效，无需额外操作
```

## 四、日常运维命令

```bash
# 查看后端服务状态
systemctl status fb-ad-logic-engine

# 查看后端实时日志
journalctl -u fb-ad-logic-engine -f

# 查看最近 50 条日志
journalctl -u fb-ad-logic-engine -n 50 --no-pager

# 查看文件日志（Winston 按日轮转，保留 14 天）
ls -la /root/work/FB-Ad-Logic-Engine/logs/
tail -100 /root/work/FB-Ad-Logic-Engine/logs/app-$(date +%Y-%m-%d).log

# 查看端口占用
ss -tlnp | grep -E "3001|80|443"

# 查看 Nginx 容器
docker ps | grep nginx
```

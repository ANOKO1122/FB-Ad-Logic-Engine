# Smart Mute（智能挂起）说明与清除方法

## Mute 是什么？

**一句话**：像「请勿打扰」——给某个广告在数据库里设一个 **mute_until（挂起截止时间）**，在这个时间之前，**所有规则**都不会对该广告执行任何动作（暂停、启用、增减预算等），只会跳过并记一条「处于 mute 期，跳过执行」。

- 数据存在 **`ad_snapshots`** 表的 `mute_until`、`mute_reason` 字段。
- 规则执行前会检查：若 `mute_until != null` 且 **当前时间 < mute_until**，则该广告被跳过。

## 干什么用？

- **人工调优**：某条广告要手动调，暂时不想被自动化改。
- **促销/活动保护**：大促期间不想规则自动暂停或改预算。
- **测试隔离**：不想某条广告被规则误动，先挂起再测别的。

## 有什么影响？

| 影响 | 说明 |
|------|------|
| 规则不执行 | 该广告匹配到规则也不会执行动作，直接跳过。 |
| 日志可见 | 日志里会看到「🔇 广告 xxx 处于 mute 期 (until ...)，跳过执行」。 |
| 摘要记录 | 若整条规则因 mute 跳过，摘要里会有 `skip_reason='muted'`、`skip_details` 里带 mute_until/mute_reason。 |
| 数据同步不覆盖 | 同步（ingestor）写入快照时**不会**覆盖 `mute_until`，所以一旦被设置会一直生效到过期或手动清除。 |

## 优缺点

| 优点 | 缺点 |
|------|------|
| 可临时排除单条广告，不影响其他广告 | 需要记得在测试完后取消 mute，否则会一直跳过 |
| 不删规则、不关规则即可生效 | 目前只能通过 DB/脚本设置或清除，前端无入口 |
| 有截止时间，可设「挂起到 19:11」自动恢复 | 若设了很远的未来时间，需手动清除才能马上恢复 |

## 怎么去掉 Mute 期？（恢复该广告被规则执行）

### 方法一：用项目自带脚本（推荐）

在项目根目录执行（把 `120236941795230501` 换成你的广告 ID）：

```bash
node server/scripts/clear-ad-mute.js 120236941795230501
```

若清除多个广告，可传多个 ID，空格分隔：

```bash
node server/scripts/clear-ad-mute.js 120236941795230501 120236941795330501
```

### 方法二：直接执行 SQL

用你的 MySQL 客户端连接同一库，执行：

```sql
UPDATE ad_snapshots
SET mute_until = NULL, mute_reason = NULL
WHERE ad_id = '120236941795230501';
```

执行后，该广告在下次规则执行时就不会再被 mute 跳过，可以正常用这条广告做测试。

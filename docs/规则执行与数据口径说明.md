# 规则执行与数据口径说明

> 本窗口相关：规则何时触发、规则判断用本地计算、前端展示用 API 数据、如何查执行/触发记录、如何验证。

---

## 1. 规则何时触发

- **仅当「Today 热数据同步完成」且该账户有数据更新时**，才会触发规则执行（AdsPolar 事件驱动）。
- 更新其他日期数据（如 daily_stats 归档）**不会**触发规则；历史数据在规则被触发后如需使用（如 last_3_days）由规则引擎查询。

详见：`server/services/ingestorService.js`（`dataUpdated` 才触发）、`server/services/cronService.js`（`executeRulesForAccount`）。

---

## 2. 规则判断口径（执行规则时）

- **CPC、ROAS、CPA、ucpc、add_to_cart_cost 等比率类指标**：一律用**本地计算**（spend/点击、purchaseValue/spend 等），分母为 0 则 0。
- **不使用**存库的 Facebook API 比值做规则判断，避免口径不一致。
- 实现位置：`server/services/ruleDataService.js` → `calculateSingleDayMetrics`。

---

## 3. 前端展示口径

- 前端展示数据时，仍可直接使用接口返回的**原始/API 数据**（如 ad_snapshots 里存的 cpc、roas），不做「一律本地计算」约束。

---

## 4. 如何查某条规则的执行/触发记录

- **接口**（需管理员）：`GET /api/rule-execution-summaries?rule_name=xxx`、`GET /api/automation-logs?rule_name=xxx`。
- **SQL**：见 `docs/按规则名查执行与触发记录.md` 第 2、4 节（规则、摘要、快照 CPC/spend/link_clicks）。

---

## 5. 如何验证「规则判断用本地计算」已生效

- **单测**：`npm test -- server/tests/ruleDataService.test.js`，关注 `calculateSingleDayMetrics` 下两条用例（link_clicks=0 时 cpc=0；有 link_clicks 时 cpc=spend/link_clicks）。
- **文档**：`docs/按规则名查执行与触发记录.md` 第 5 节（含手动验证步骤）。

## 二、手工验证关键场景

### 2.1 预算动作类型 × 执行模式矩阵

| 动作类型 | Dry Run | 真实执行 | 验收要点 |
|----------|---------|----------|----------|
| **increase_budget (percent)** | ✓ | ✓ | 日志 newBudgetCents 正确；FB 后台预算变化 |
| **increase_budget (usd)** | ✓ | ✓ | 同上；api_request 含 value_unit=usd |
| **decrease_budget (percent)** | ✓ | ✓ | 下限护栏 100 分；低于 100 时结果为 100 |
| **decrease_budget (usd)** | ✓ | ✓ | 同上 |
| **set_budget** | ✓ | ✓ | 不依赖当前预算；CBO Dry Run 时 newBudgetCents 非 0 |

### 2.2 ABO 场景（广告组有预算）

1. 选一条 **广告组有日预算** 的广告，建规则：
   - 条件：spend > 0（today）
   - 动作：增加预算 10%（percent）或 +$5（usd）或 设置预算 $30（set_budget）
2. **Dry Run**：点「运行此规则」→ 日志应有 `广告组`、`newBudgetCents` 非 0
3. **真实执行**：关闭 Dry Run → 再运行 → FB 后台该广告组预算应变化
4. **审计**：查 automation_logs 最新一条，`api_request` / `api_response` 含 `budgetTarget: "广告组"`

### 2.3 CBO 场景（系列预算）

1. 选一条 **只有系列预算、广告组无预算** 的广告，建规则：
   - 同上条件与动作
2. **Dry Run + set_budget**：
   - 动作选「设置预算为 $30」
   - 运行 → **关键**：`api_response` 中 `newBudgetCents` 应为 3000，**不为 0**
3. **Dry Run + increase/decrease**：`newBudgetCents` 可为 0（不拉取当前预算），属预期
4. **真实执行**：关闭 Dry Run → 运行 → FB 后台该**广告系列**预算应变化
5. **审计**：`api_request` / `api_response` 含 `budgetTarget: "广告系列(CBO)"`

### 2.4 上限与下限

| 场景 | 操作 | 预期 |
|------|------|------|
| **上限护栏** | set_budget $80，预算上限填 50 美元（5000 分） | 结果 5000 分 |
| **cap 有效化** | （若绕过前端）max_daily_budget=0 | 执行层结果 100 分（capEffective） |
| **下限护栏** | decrease_budget 导致 < 100 分 | 结果 100 分 |
| **校验阻断** | 新建规则时预算上限填 0.5 美元 | 后端保存失败：`max_daily_budget 须 >= 100 分` |

### 2.5 前端归一化

1. 手工改库：`UPDATE rules SET actions = '[{"type":"increase_budget","value":10,"max_daily_budget":50}]' WHERE id=某规则`
2. 前端点「编辑」该规则
3. 控制台应有：`[规则编辑] 已自动清空 1 个无效的预算上限（<1美元）`
4. 表单中预算上限输入框应为空（已清空）

### 2.6 模板同场景

- AdminTemplates：编辑含 `max_daily_budget < 100` 的模板 → 同上控制台与清空
- 保存模板时，预算上限 min=1，输入 0 不传 max_daily_budget

## 三、automation_logs 审计字段验收

### 3.1 必含字段（每条预算动作记录）

| 字段 | 含义 | 验收 |
|------|------|------|
| `run_id` | M4 运行批次 | 单条规则执行时有值 |
| `account_id` | 广告账户 | 非空 |
| `ad_id` | 广告 ID | 非空 |
| `rule_id` / `rule_name` | 规则标识 | 非空 |
| `action_type` | 动作类型 | INCREASE_BUDGET / DECREASE_BUDGET / SET_BUDGET |
| `is_simulation` | 是否 Dry Run | 与规则一致 |
| `status` | success / fail / skipped | 非空 |
| `metrics_snapshot` | 触发时指标 | 含 spend、roas、cpa 等，便于复盘 |
| `action_payload` | 动作参数 | 含 type、value、value_unit、max_daily_budget |
| `api_request` | 请求快照 | 含 budgetTarget、adjustType、value_unit、adjustValue、**newBudgetCents** |
| `api_response` | 响应快照 | 含 success、**newBudgetCents**、change、budgetTarget |
| `triggered_at` | 触发时间 | 非空，UTC |

### 3.2 预算动作专用检查（api_request / api_response JSON）

**api_request 应含**：
- `budgetTarget`：`"广告组"` 或 `"广告系列(CBO)"`
- `adjustType`：increase_budget / decrease_budget / set_budget
- `value_unit`：percent / usd
- `adjustValue`：数值
- `newBudgetCents`：**CBO + Dry Run + set_budget 时不应为 0**

**api_response 应含**：
- `newBudgetCents`：同上
- `change`：如 `+10%`、`+$5`、`=$30`
- `budgetTarget`：与 api_request 一致

### 3.3 验收 SQL（预算动作最近 10 条）

```sql
SELECT
  id, run_id, account_id, ad_id, rule_name, action_type, is_simulation, status,
  JSON_EXTRACT(api_request, '$.newBudgetCents') AS req_new_cents,
  JSON_EXTRACT(api_response, '$.newBudgetCents') AS res_new_cents,
  JSON_EXTRACT(api_request, '$.budgetTarget') AS budget_target,
  triggered_at
FROM automation_logs
WHERE action_type IN ('INCREASE_BUDGET', 'DECREASE_BUDGET', 'SET_BUDGET')
ORDER BY id DESC
LIMIT 10;
```

**预期**：
- `req_new_cents` / `res_new_cents` 在 set_budget 或真实执行时非 0（CBO Dry Run 的 increase/decrease 可为 0）
- `budget_target` 为 "广告组" 或 "广告系列(CBO)"

# M4 3.5 预算单位统一为「分」— 验收步骤

> 后端预算一律整数「分」。FB API 的 daily_budget 读写均为「分」：getAdsetBudget 直接返回 API 值（不再乘 100），updateAdsetBudget 直接传分。前端若有预算展示需除以 100。

---

## 1. 单测验证（你本机执行）

在项目根目录执行：

```bash
npm test -- --run server/tests/budgetUnitCents.test.js
```

**预期**：全部用例通过（computeNewBudgetCentsOnce 纯函数、getAdsetBudget 返回美分、updateAdsetBudget 接受美分、端到端美分换算）。

---

## 2. 后端口径核对（可选）

- **actionExecutorService**：`MIN_BUDGET_CENTS = 100`，`computeNewBudgetCentsOnce` 入参/返回均为分；日志为「xxx 分」。
- **server/index.js**：`getAdsetBudget` 将 FB 返回的 daily_budget（已是分）直接取整返回；`updateAdsetBudget(adsetId, newBudgetCents)` 直接传分给 FB（daily_budget 为最小货币单位）。
- **schema**：`rules` 或动作相关注释中注明 `max_daily_budget` 单位为「分」（已存在于 server/db/schema.js）。

---

## 3. 前端（若有预算展示）

若规则配置或日志里有「预算金额」展示，应为「分 ÷ 100」显示为元/美元。当前 RuleManager 仅动作类型（增加/减少预算），未展示具体预算数值时可跳过。

---

## 4. 验收通过后

在 TASKS.md 的「3.5 步骤 5」中勾选各项，并注明「本机单测通过 + 后端口径已统一为分」。

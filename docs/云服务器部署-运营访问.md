# 云服务器部署：运营可浏览器访问

目标：**前端页面 + `/api` 同源**（与本地 Vite `proxy` 行为一致），**Node 常驻**，**防火墙放行**。

假设项目路径为 `/root/work/FB-Ad-Logic-Engine`，对外 Web 端口 **`8088`**（与融合平台占用的 **80** 不冲突，可按需改）。

---

## 阶段 1：代码与构建

```bash
cd /root/work/FB-Ad-Logic-Engine
git pull
npm install
npm run build
```

确认存在目录 **`dist/`**。

---

## 阶段 2：环境变量（`.env`）

1. 配置 **`DB_*`**、**`FACEBOOK_ACCESS_TOKEN`**、**`PORT=3001`**、代理等（与当前环境一致）。
2. **云服务器**上建议取消注释：

   ```env
   NODE_ENV=production
   ```

   这样会写入项目下 **`logs/`**（约 14 天轮转），便于排障。

---

## 阶段 3：Node 常驻（systemd，推荐）

1. 确认 Node 路径：

   ```bash
   which node
   ```

2. 编辑服务文件中的 **`WorkingDirectory`**、**`EnvironmentFile`**、**`ExecStart`**（若 `node` 不在 `/usr/bin/node`）：

   ```bash
   sudo cp deploy/fb-ad-logic-engine.service /etc/systemd/system/fb-ad-logic-engine.service
   sudo nano /etc/systemd/system/fb-ad-logic-engine.service
   ```

3. 启用并启动：

   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable --now fb-ad-logic-engine
   sudo systemctl status fb-ad-logic-engine
   ```

4. 验证 API：

   ```bash
   curl -sS http://127.0.0.1:3001/api/health
   ```

查看日志：`journalctl -u fb-ad-logic-engine -f`；文件日志：`tail -f logs/combined-*.log`。

---

## 阶段 4：对运营开放网页（二选一）

### 方式 A：独立 Nginx 容器（不改融合平台 compose）

把 **`8088`** 映射到容器 **80**，静态目录挂载 **`dist`**，API 通过 **`host.docker.internal`** 访问宿主机 **3001**。

```bash
cd /root/work/FB-Ad-Logic-Engine

docker rm -f fb-ad-nginx 2>/dev/null || true

docker run -d --name fb-ad-nginx --restart unless-stopped \
  -p 8088:80 \
  -v "$(pwd)/dist:/usr/share/nginx/html:ro" \
  -v "$(pwd)/deploy/nginx-fb-ad-docker.conf:/etc/nginx/conf.d/default.conf:ro" \
  --add-host=host.docker.internal:host-gateway \
  nginx:alpine
```

浏览器访问：`http://<服务器公网IP>:8088/`  
（需防火墙/安全组放行 **8088/TCP**。）

**排障**：若页面能打开但接口 **502**，先在**宿主机**确认 API 已启动：

```bash
curl -sS http://127.0.0.1:3001/api/health
```

正常后再测经 Nginx：`curl -sS http://127.0.0.1:8088/api/health`。  
Nginx 容器依赖 **`--add-host=host.docker.internal:host-gateway`** 才能访问宿主机上的 Node。

### 方式 B：宿主机已装 Nginx

```bash
sudo cp deploy/nginx-fb-ad-host.example.conf /etc/nginx/conf.d/fb-ad-logic.conf
# 修改 root 路径、listen 端口后：
sudo nginx -t && sudo systemctl reload nginx
```

## 阶段 5：防火墙（EL9 示例）

```bash
sudo firewall-cmd --permanent --add-port=8088/tcp
sudo firewall-cmd --reload
```

云厂商安全组同样放行 **8088**。

## 与融合平台（Docker Compose）的关系

- **80 端口**：当前由 FB 入口容器 **`fb-ad-nginx`** 提供。
- **融合平台**：继续使用 **`8081`**（`ronghe-nginx`）并保留过渡入口。
- **MySQL**：融合平台 Compose 中容器名已统一为 **`mysql`**（例如 `docker exec -it mysql mysql -uroot -p…`）。FB 侧 `.env` 仍可用 **`DB_HOST=127.0.0.1`**、`3306`（经宿主映射连库，与当前一致）。

---

## 钉钉认证预留（里程碑 3 骨架）

- 已在两侧 Nginx 预留 `/oauth2/` 反向代理与 `/oauth2/auth` 子请求端点，后续可直接切到 `auth_request`。
- 回调地址固定为：`http://ad_tools.thinkpro.top/oauth2/callback`。
- 钉钉开放平台需预留出口 IP 白名单：`104.194.95.40`。
- 在未填 `AppKey/AppSecret` 前，先不要启用强制认证拦截，避免入口被锁死。
- 认证头约定统一为：`X-Forwarded-User`，后端按 `req.headers['x-forwarded-user']` 读取登录身份。

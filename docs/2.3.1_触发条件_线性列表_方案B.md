# 2.3.1 触发条件线性列表（方案B）

> 目标：RuleManager 触发条件改为线性条件列表，第 2 行开始每行可选 AND/OR；去掉全局「满足所有/任一」选择；新增全局 time_window/custom_range；保存时把全局窗口写进每条 condition，并把线性 AND/OR 转成 v2 DNF groups。后端保持不变。

## 一、方案B 约束

- **全局时间窗口**：同一条规则所有条件共用同一个 time_window / custom_range。
- **线性 AND/OR 列表**：UI 每行一条条件；第 2 行开始左侧显示 AND/OR 下拉，表示该条件与上面所有条件的逻辑关系。
- **转换规则**：AND → 追加到当前组；OR → 开启新组。保存时输出 v2 DNF：`{ version: 2, groups: [...] }`。

### 优缺点

| | 说明 |
|---|------|
| **优点** | 后端几乎不用动；规则只查一个时间窗口，性能最好，结果更好解释 |
| **缺点** | 不能在同一条规则里混用「今天 + 近3天」等不同窗口，需拆成多条规则 |

---

## 二、阶段拆分（4 阶段）

| 阶段 | 目标 | 验收标准 |
|------|------|----------|
| **阶段 1** | 转换函数 + 初始化逻辑 | 转换函数输出正确；空列表自动补 1 行；首行 join=null；custom_range 默认日期 |
| **阶段 2** | UI：全局时间窗口 + 线性列表 | 顶部 time_window 下拉 + custom_range 日期；移除 logicOperator；第 2 行起 AND/OR 下拉 |
| **阶段 3** | 保存与回填 | 保存输出 v2、每条 condition 同 time_window/custom_range；回填 AND/OR 正确 |
| **阶段 4** | 模板适配 | 点模板后填充 whenLines + whenTimeWindow，移除 v2 JSON 模式 |

---

## 三、必须验收点

### 保存

- 保存的 conditions 必须是 v2：`{ version: 2, groups: [...] }`
- 每条 condition 必须带相同的 `time_window`
- 若为 custom_range，每条必须带相同的 `custom_range.since` / `custom_range.until`（否则后端 400 或执行报错）

### 回填

- 从 v2 任意一条 condition 读出 time_window / custom_range，设置到全局控件
- 线性 join 的 OR 必须落在「新 group 的第一条」上，避免 OR 丢失

### 边界

- 删到 0 条时自动补 1 条
- 第 2 行开始才能选 OR
- 第 1 行 join 永远为 null，UI 不显示

## 四、转换函数伪代码

### whenLines 单行结构

```javascript
{ join: 'AND' | 'OR' | null, metric, operator, value }
// 首行 join 必为 null；第 2 行起 join 为 'AND' 或 'OR'
```

### linesToV2Groups(whenLines, whenTimeWindow, whenCustomRange)

```
groups = []
currentGroup = []
for each line in whenLines:
  cond = { metric, operator, value, time_window: whenTimeWindow }
  if whenCustomRange: cond.custom_range = whenCustomRange
  
  if line.join === 'OR' or (line.join === null and currentGroup.length > 0):
    // OR：开启新组；若有当前组先 push
    if currentGroup.length > 0:
      groups.push({ operator: 'AND', conditions: currentGroup })
    currentGroup = [cond]
  else:
    // AND 或首行
    currentGroup.push(cond)
if currentGroup.length > 0:
  groups.push({ operator: 'AND', conditions: currentGroup })
return { version: 2, groups }
```

### v2ToLines(v2)

```
lines = []
for i, group in enumerate(v2.groups):
  for j, cond in enumerate(group.conditions):
    join = (i === 0 and j === 0) ? null : (j === 0 ? 'OR' : 'AND')
    lines.push({ join, metric: cond.metric, operator: cond.operator, value: cond.value })
return lines
// 全局 time_window/custom_range 从任意 cond 取
```

### v1ToLines(v1Array, logicOperator)

```
lines = v1Array.map((cond, i) => ({
  join: i === 0 ? null : logicOperator,
  metric: cond.metric,
  operator: cond.operator,
  value: cond.value
}))
return lines
// 全局 time_window/custom_range 从第一条 cond 取（无则 today）
```

## 六、阶段 1 控制台验证

开发环境下，RuleManager 会将转换函数挂到 `window.__ruleManagerVerify`。打开规则管理页面后，在浏览器控制台执行：

```javascript
const { linesToV2Groups, v2ToLines, v1ToLines } = window.__ruleManagerVerify

// 1. 线性 → v2：A AND B OR C
const lines = [
  { join: null, metric: 'spend', operator: 'gt', value: 20 },
  { join: 'AND', metric: 'roas', operator: 'lt', value: 0.5 },
  { join: 'OR', metric: 'cpc', operator: 'gt', value: 0.8 }
]
const v2 = linesToV2Groups(lines, 'today', null)
console.log(JSON.stringify(v2, null, 2))
// 预期：groups 2 个，第 1 组 2 条，第 2 组 1 条；每条都有 time_window:'today'

// 2. v2 → 线性（回填）
const back = v2ToLines(v2)
console.log(back.lines)
// 预期：lines[0].join===null, lines[1].join==='AND', lines[2].join==='OR'

// 3. v1 → 线性
const v1Back = v1ToLines([{ metric: 'spend', operator: 'gt', value: 10, time_window: 'today' }], 'OR')
console.log(v1Back)
// 预期：lines[0].join===null, timeWindow==='today'
```

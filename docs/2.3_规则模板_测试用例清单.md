# 2.3 规则模板 - 测试用例清单

> 可交给测试同学执行，覆盖 admin/普通用户两种角色。
> 更新：2026-02-06 - P0 空耗模板已实现；贵点击/空转化为 P1。

## 前提（强约束）

1. **先 Dry Run，再真执行**：同一条规则先 Dry Run 确认命中数量、字段无误，再关闭 Dry Run。
2. **缩小作用范围**：测试规则只选 1–3 个 ad 作为 target_ids，不要全账户跑。
3. **低风险阈值**：如 spend > 0.5 AND link_clicks = 0，确保这 1–3 个广告确实符合条件。
4. **审计验证**：每次测试后查 automation_logs 的 rule_id, ad_id, action_type, is_simulation, status, error_message, metrics_snapshot, run_id，确保可复盘。

## 管理员（admin）用例

| 用例 | 操作 | 期望 |
|------|------|------|
| **1. AND 组合** | 创建规则：条件① spend > 0.5、条件② link_clicks = 0，逻辑 AND，动作 pause_ad；选 1–2 个符合条件广告；先 Dry Run 再真执行 | Dry Run 命中预期广告；真执行后 automation_logs 有 success 记录；字段完整 |
| **2. 空耗模板** | 点击「空耗模板」，确认条件为 spend > 0.5 AND link_clicks = 0；选 1–2 个符合条件广告；先 Dry Run 再真执行 | 模板正确填充；Dry Run/真执行命中与用例 1 一致；automation_logs 可复盘 |
| **3. 贵点击模板** | （P1 已实现后）点击「贵点击模板」，条件为 link_clicks > 0 AND cpc > Y；选符合条件广告；Dry Run → 真执行 | 同上 |

## 普通用户（non-admin）用例

| 用例 | 操作 | 期望 |
|------|------|------|
| **1. 有权限账户** | 用普通用户创建/执行规则，target 为该用户 owner_id 下的广告 | 能创建、保存、执行；automation_logs 有记录；执行日志可见 |
| **2. 无权限账户** | 尝试对非该用户负责的账户执行规则（如篡改接口参数） | **手动执行接口**：直接返回 403，不写 automation_logs。<br>**定时/批量执行**：rule_execution_summaries 中 skip_reason=no_permission（automation_logs 不写）<br>※ no_permission 只在执行摘要里出现，手动接口无权限是 403 |
| **3. AND/OR 组合** | 同管理员用例 1，用普通用户账号 | 行为与管理员一致；仅限其负责的账户 |

## 验收 SQL

```sql
SELECT rule_id, ad_id, action_type, is_simulation, status, error_message, 
       JSON_EXTRACT(metrics_snapshot, '$.spend') as spend,
       JSON_EXTRACT(metrics_snapshot, '$.link_clicks') as link_clicks,
       run_id, triggered_at
FROM automation_logs
ORDER BY id DESC
LIMIT 5;
```

单条记录应能回答：谁、何时、对哪个广告、做了何动作、结果如何、当时 spend/link_clicks 等。

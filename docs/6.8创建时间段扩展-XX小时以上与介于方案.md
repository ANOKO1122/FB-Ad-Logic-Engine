# 动态筛选「创建时间段」扩展方案 — XX小时以上 + 介于XX~YY小时之间

## 一、需求

在规则编辑 → 动态筛选 → 监控范围条件的「创建时间段」中，新增两种时间筛选模式：

| 新模式 | 语义 | 示例 |
|--------|------|------|
| **XX 小时以上** | 创建时间距今超过 X 小时 | 创建超过 48 小时的广告 |
| **介于 XX~YY 小时之间** | 创建于 X 小时以上（超过 X 小时），但在 Y 小时之内（未超过 Y 小时） | 创建超过 24 小时但在 72 小时之内的广告 |

原有「近 N 小时内」保持不变（创建时间在最近 N 小时内）。

---

## 二、现状回顾

### 2.1 当前值下拉选项

| 选项 | 含义 |
|------|------|
| 近 24 小时内 | within_hours=24 |
| 近 48 小时内 | within_hours=48 |
| 近 72 小时内 | within_hours=72 |
| 自定义小时内 | within_hours=N（用户填 1-720） |

### 2.2 当前前后端数据结构

**前端 row 对象**（`scopeConditionRows`）：
```js
{
  id: 'scope-xxx',
  field: 'created_within',
  operator: 'include',        // 固定
  value: '24' | '48' | '72' | 'custom',
  customHours: null | number  // value='custom' 时使用
}
```

**后端序列化 → scopeFilters.conditions**：
```json
{ "field": "created_time", "operator": "within_hours", "value": 48 }
```

**后端 SQL**（`dynamicScopeService.js` → `buildStructureFilter`）：
```sql
created_time IS NOT NULL AND created_time >= ?   -- ? = now - N小时
```

### 2.3 当前匹配的数据源

`structure_ads.created_time` / `structure_adsets.created_time` / `structure_campaigns.created_time`

这是 Facebook API 返回的**广告/广告组/广告系列创建时间**（ISO8601），**不是**首次产生花费的时间。

**改为**：
```js
    } else if (field === 'created_time') {
      if (op === 'within_hours' || !op) {
        // 现有：最近 N 小时内 → created_time >= now - N hours
        const hours = Number(val)
        if (!Number.isFinite(hours) || hours <= 0) continue
        const threshold = now.minus({ hours }).toUTC().toISO()
        clauses.push('created_time IS NOT NULL AND created_time >= ?')
        params.push(threshold)
      } else if (op === 'older_than_hours') {
        // 新增：XX 小时以前 → created_time <= now - N hours
        const hours = Number(val)
        if (!Number.isFinite(hours) || hours <= 0) continue
        const threshold = now.minus({ hours }).toUTC().toISO()
        clauses.push('created_time IS NOT NULL AND created_time <= ?')
        params.push(threshold)
      } else if (op === 'between_hours') {
        // 新增：X 小时以上（超过 X）、Y 小时以内（未超过 Y）
        // value 为 [fromHours, toHours]，fromHours < toHours
        // 例: [24, 72] → created_time BETWEEN now-72h AND now-24h
        const arr = Array.isArray(val) ? val : []
        const fromHours = Number(arr[0])
        const toHours = Number(arr[1])
        if (!Number.isFinite(fromHours) || !Number.isFinite(toHours) || fromHours <= 0 || toHours <= 0 || fromHours >= toHours) continue
        const lowerBound = now.minus({ hours: toHours }).toUTC().toISO()    // 更早: now - 72h
        const upperBound = now.minus({ hours: fromHours }).toUTC().toISO()  // 更近: now - 24h
        clauses.push('created_time IS NOT NULL AND created_time >= ? AND created_time <= ?')
        params.push(lowerBound, upperBound)
      } else {
        throw new Error('INVALID_FILTER_OPERATOR_CREATED_TIME')
      }
    } else {
      throw new Error('INVALID_FILTER_FIELD')
    }
```

### 5.2 SQL 对应关系

| 前端选项 | operator | SQL |
|---------|----------|-----|
| 近 N 小时内 | `within_hours` | `created_time >= now - N hours` |
| XX 小时以前 | `older_than_hours` | `created_time <= now - N hours` |
| XX 小时以上、YY 小时以内 | `between_hours` | `created_time >= now - YY AND created_time <= now - XX` |

### 5.3 向后兼容

现有数据库中已存储的 `scope_filters`，其 `created_time` 条件均不带 `operator` 字段（历史数据走 `!op` 分支），代码中通过 `op === 'within_hours' || !op` 兜底，**向后完全兼容**旧数据。

#### 5.4.2 `server/services/structureSyncService.js` — 4 个 list 函数新增时间过滤

涉及函数：`listStructureAdsFromDb`、`listStructureCampaignsFromDb`、`listStructureAdsetsFromDb`、`listStructureObjectsFromDb`

**a) 三表 list 函数（ads / campaigns / adsets）**：在现有 `createdSinceThreshold` 计算之后追加：

```js
  // 新增：创建超过 N 小时 → created_time <= now - N hours
  const rawBeforeHours = opts.scope_created_before_hours != null ? parseInt(opts.scope_created_before_hours, 10) : NaN
  const beforeHours = Number.isFinite(rawBeforeHours) && rawBeforeHours > 0 ? rawBeforeHours : null
  const createdBeforeThreshold = beforeHours != null ? new Date(Date.now() - beforeHours * 3600 * 1000).toISOString() : null

  // 新增：介于 X~Y 小时之间（超过 X、未超过 Y）
  // fromHours < toHours；例 from=24, to=72 → created_time BETWEEN now-72h AND now-24h
  const rawBetweenFrom = opts.scope_created_between_from_hours != null ? parseInt(opts.scope_created_between_from_hours, 10) : NaN
  const rawBetweenTo = opts.scope_created_between_to_hours != null ? parseInt(opts.scope_created_between_to_hours, 10) : NaN
  const betweenFromHours = Number.isFinite(rawBetweenFrom) && rawBetweenFrom > 0 ? rawBetweenFrom : null
  const betweenToHours = Number.isFinite(rawBetweenTo) && rawBetweenTo > 0 ? rawBetweenTo : null
  const betweenValid = betweenFromHours != null && betweenToHours != null && betweenFromHours < betweenToHours
  const betweenLowerBound = betweenValid ? new Date(Date.now() - betweenToHours * 3600 * 1000).toISOString() : null    // 更早: now - Y
  const betweenUpperBound = betweenValid ? new Date(Date.now() - betweenFromHours * 3600 * 1000).toISOString() : null  // 更近: now - X
```

在现有 `if (createdSinceThreshold)` SQL 块之后追加：

```js
  if (createdBeforeThreshold) {
    sql += ` AND created_time IS NOT NULL AND created_time <= ?`
    params.push(createdBeforeThreshold)
  }
  if (betweenValid) {
    sql += ` AND created_time IS NOT NULL AND created_time >= ? AND created_time <= ?`
    params.push(betweenLowerBound, betweenUpperBound)
  }
```

**b) `listStructureObjectsFromDb`**：opts 透传 3 个新字段：

```js
  const listOpts = {
    // ...现有字段...
    scope_created_within_hours: opts.scope_created_within_hours,
    scope_created_before_hours: opts.scope_created_before_hours,          // 新增
    scope_created_between_from_hours: opts.scope_created_between_from_hours, // 新增
    scope_created_between_to_hours: opts.scope_created_between_to_hours      // 新增
  }
```

**c) JSDoc 更新**：每个 list 函数的 `@param {Object} opts` 注释追加：

```
 *   scope_created_before_hours         - 创建超过 N 小时（older_than）
 *   scope_created_between_from_hours   - 介于起始小时（超过 X）
 *   scope_created_between_to_hours     - 介于结束小时（Y 以内）
```

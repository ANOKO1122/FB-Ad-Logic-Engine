## 二、数据库验证

### 2.1 连接数据库

使用 MySQL 客户端（如 MySQL Workbench、命令行、DBeaver）连接：

```powershell
mysql -u root -p fb_ad_brain
```

### 2.2 检查表与数据

```sql
-- 1. 表是否存在
SHOW TABLES LIKE 'rule_templates';

-- 2. 查看 3 条预设模板
SELECT id, name, slug, when_time_window, sort_order, is_active
FROM rule_templates
ORDER BY sort_order;

-- 预期结果：3 行
-- id=1 止损 today 10 1
-- id=2 扩量 last_3_days 20 1
-- id=3 空耗 today 30 1
```

### 2.3 验证 slug 唯一约束

```sql
-- 应报错：Duplicate entry 'stop_loss' for key 'uk_slug'

VALUES ('重复测试', 'stop_loss', '[]', 'today', '[]');
```

若报错，则唯一约束正常。

## 三、接口验证

### 3.1 获取 Token

1. 浏览器访问 `http://localhost:5173/login`
2. 用管理员账号登录
3. 打开开发者工具（F12）→ Application → Cookies
4. 找到 `token`，复制其值（或从 Network 中任一请求的 Cookie 里复制）

### 3.2 使用 Thunder Client（VS Code 插件）

若已安装 Thunder Client，可新建请求集合进行测试。

#### 3.2.1 GET /api/templates（普通用户可用）

- **Method**：GET  
- **URL**：`http://localhost:3000/api/templates`  
- **Headers**：无（Cookie 会自动携带，或手动加 `Cookie: token=你的token`）  
- 或在浏览器登录后，直接访问：`http://localhost:3000/api/templates`，查看 JSON 返回

**预期**：`{ "success": true, "templates": [ ... ] }`，且只含 `is_active=1` 的模板。

#### 3.2.2 GET /api/admin/templates（管理员）

- **Method**：GET  
- **URL**：`http://localhost:3000/api/admin/templates?include_inactive=1&limit=20&offset=0`  
- **Headers**：`Cookie: token=你的管理员token`  

**预期**：返回所有模板（含已删除）。

#### 3.2.3 POST /api/admin/templates（新建）

- **Method**：POST  
- **URL**：`http://localhost:3000/api/admin/templates`  
- **Headers**：`Content-Type: application/json`、`Cookie: token=你的管理员token`  
- **Body**（raw JSON）：

```json
{
  "name": "验证用模板",
  "slug": "verify_test",
  "description": "用于接口验证",
  "sort_order": 99,
  "when_lines": [
    { "join": null, "metric": "spend", "operator": "gt", "value": 5 }
  ],
  "when_time_window": "today",
  "when_custom_range": null,
  "actions": [ { "type": "pause_ad", "value": null } ]
}
```

**预期**：201，返回 `{ "success": true, "template": { "id": 4, ... }, "message": "模板创建成功" }`

#### 3.2.4 PUT /api/admin/templates/:id（更新，需 updated_at）

先从 GET 列表里拿到刚创建模板的 `updated_at`（如 `"2025-02-12T10:30:00.000Z"`）。

- **Method**：PUT  
- **URL**：`http://localhost:3000/api/admin/templates/4`  
- **Headers**：`Content-Type: application/json`、`Cookie: token=你的管理员token`  
- **Body**：

```json
{
  "name": "验证用模板-已改",
  "description": "已更新描述",
  "sort_order": 99,
  "when_lines": [
    { "join": null, "metric": "spend", "operator": "gt", "value": 10 }
  ],
  "when_time_window": "today",
  "when_custom_range": null,
  "actions": [ { "type": "pause_ad", "value": null } ],
  "updated_at": "这里填上一步获取的 updated_at"
}
```

**预期**：200，模板更新成功。

**乐观并发测试**：把 `updated_at` 改成错误值（如 `"2020-01-01T00:00:00.000Z"`），再发一次 PUT。  
**预期**：409，`{ "error": "模板已被他人修改，请刷新后重试", "code": "CONFLICT" }`

#### 3.2.5 DELETE /api/admin/templates/:id（软删除）

- **Method**：DELETE  
- **URL**：`http://localhost:3000/api/admin/templates/4`  
- **Headers**：`Cookie: token=你的管理员token`  

**预期**：200，`{ "success": true, "message": "模板「xxx」已删除（软删除）..." }`

再 GET `/api/templates`，应不再包含 id=4；GET `/api/admin/templates?include_inactive=1` 仍包含，且 `is_active: false`。

#### 3.2.6 POST /api/admin/templates/:id/restore（恢复）

- **Method**：POST  
- **URL**：`http://localhost:3000/api/admin/templates/4/restore`  
- **Headers**：`Cookie: token=你的管理员token`  

**预期**：200，`{ "success": true, "message": "模板「xxx」已恢复" }`

再 GET `/api/templates`，应重新看到该模板。

#### 3.2.7 越权验证（普通用户调用 admin 接口）

1. 用**普通用户**登录，复制其 token  
2. 用该 token 发：`POST http://localhost:3000/api/admin/templates`  
3. Body 同上新建模板的 JSON  

**预期**：403，`{ "error": "需要管理员权限", "code": "ADMIN_REQUIRED" }`

### 3.3 使用 curl（命令行）

将 `YOUR_TOKEN` 替换为实际 token：

```powershell
# GET 模板列表
curl -X GET "http://localhost:3000/api/templates" -H "Cookie: token=YOUR_TOKEN"

# 管理员列表
curl -X GET "http://localhost:3000/api/admin/templates?include_inactive=1" -H "Cookie: token=YOUR_TOKEN"
```

## 四、前端验证

### 4.1 普通用户

1. 用**普通用户**登录  
2. 查看侧边栏：**不应**出现「模板管理」  
3. 进入「规则管理」→ 点击「新建规则」  
4. 在「3. 触发条件」区域：应看到模板按钮（如「止损」「扩量」「空耗」）  
5. 点击「止损」→ 弹出确认框「这将覆盖当前条件设置，确定吗？」→ 点确定  
6. 检查：时间窗口应为「今天」，条件行应有「花费 > 20」「ROAS < 0.5」，动作为「暂停广告」  
7. 填写规则名称、选择账户等，保存规则  
8. 再次编辑该规则，确认 conditions/actions 与模板一致  

### 4.2 管理员 — 模板管理页

1. 用**管理员**登录  
2. 侧边栏应出现「模板管理」，点击进入  
3. 应看到卡片列表（至少 3 个预设模板）  

#### 4.2.1 新建模板

1. 点击「+ 新建模板」  
2. 填写：  
   - 名称：`测试新建`  
   - slug：`test_new`  
   - 描述：`验证新建`  
   - 排序：`50`  
3. 条件：保留默认或改为「花费 > 10」  
4. 动作：保留「暂停广告」  
5. 点击「保存」  
6. 列表应多出一张新卡片  

#### 4.2.2 编辑模板

1. 点击某张卡片的「编辑」  
2. 修改名称或描述  
3. 注意：slug 输入框应为只读（灰色或不可编辑）  
4. 点击「保存」  
5. 列表中该卡片内容应已更新  

#### 4.2.3 删除与恢复

1. 点击「测试新建」卡片的「删除」  
2. 确认弹窗 → 确定  
3. 若勾选了「显示已删除」，卡片仍可见但标为「已删除」，且出现「恢复」按钮  
4. 若未勾选「显示已删除」，该卡片从列表消失  
5. 勾选「显示已删除」，点击「恢复」  
6. 卡片恢复为正常状态，可再次在规则管理中使用  

#### 4.2.4 预算动作与单位

1. 新建或编辑模板，动作选「增加预算」  
2. 填写百分比：`20`  
3. 在「预算上限」输入 `100`（美元）  
4. 查看输入框旁提示：「美元（保存时转为分存储）」  
5. 保存后，数据库中 `actions` 的 `max_daily_budget` 应为 `10000`（100 美元 = 10000 分）  

### 4.3 规则管理中使用模板

1. 管理员或普通用户进入「规则管理」→「新建规则」  
2. 选择账户、填写规则名称  
3. 点击任一模板按钮（如「扩量」）  
4. 确认后，条件、时间窗口、动作应自动填充  
5. 保存规则  
6. 编辑该规则，确认数据与模板一致

## 六、常见问题

### Q1：GET /api/templates 返回 401

**原因**：未登录或 token 失效。  
**处理**：重新登录，确保请求带 Cookie 或 Authorization。

### Q2：PUT 一直返回 409

**原因**：`updated_at` 与数据库不一致（可能编辑前数据已变更）。  
**处理**：关闭弹窗，刷新列表，重新点「编辑」再保存。

### Q3：规则管理页没有模板按钮

**原因**：GET /api/templates 失败或返回空数组。  
**处理**：检查 Network 中 `/api/templates` 的返回；确认种子数据已插入且 `is_active=1`。

### Q4：删除后想用同一 slug 新建

**原因**：slug 全局唯一，软删除后仍占用。  
**处理**：使用「恢复」接口或恢复按钮，而不是新建同 slug 的模板。

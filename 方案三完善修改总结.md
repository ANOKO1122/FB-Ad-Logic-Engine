## 🔧 已完成的完善修改

### 1. Drizzle Schema：accountId 设为 NOT NULL ✅

**文件**：`server/db/schema.js`

**修改前**：
```javascript
// 如果为 null，则规则作用于用户可见的所有账户（向后兼容）
accountId: varchar('account_id', { length: 50 }),
```

**修改后**：
```javascript
// ✅ 方案三：必须绑定账户（NOT NULL），防止反向索引退化
accountId: varchar('account_id', { length: 50 }).notNull(),
```

**影响**：
- ✅ 代码语义与数据库约束一致
- ✅ 防止未来通过其他入口创建规则时漏传 accountId

---

### 2. RulesService：移除 accountId 的默认值 ✅

**文件**：`server/services/rulesService.js`

**修改前**：
```javascript
// 广告账户 ID：限定规则只作用于此账户（如果为 null，则作用于用户可见的所有账户）
accountId: ruleData.accountId || null,
```

**修改后**：
```javascript
// ✅ 方案三：accountId 必填（路由层已校验，这里直接使用）
// 如果路由层校验通过，accountId 一定存在且非空
accountId: ruleData.accountId,
```

**影响**：
- ✅ 不再主动写入 NULL
- ✅ 与数据库 NOT NULL 约束一致
- ✅ 如果路由层校验通过，accountId 一定存在

---

### 3. 路由层：防御性类型转换 ✅

**文件**：`server/routes/rules.js`

**修改前**：
```javascript
if (!accountId || accountId.trim() === '') {
  // 如果 accountId 是数字/对象，.trim() 会抛异常
}
```

**修改后**：
```javascript
// 使用 String() 防御非字符串类型，避免 .trim() 抛异常
const accountIdStr = accountId != null ? String(accountId).trim() : ''
if (!accountIdStr) {
  return res.status(400).json({ 
    error: '缺少必填字段: accountId（请选择广告账户）',
    code: 'MISSING_ACCOUNT_ID'
  })
}
```

**影响**：
- ✅ 防止非字符串类型导致 `.trim()` 抛异常
- ✅ 更健壮的类型处理
- ✅ POST 和 PUT 路由都已修改

### 4. PUT 路由：补充校验覆盖 ✅

**文件**：`server/routes/rules.js`

**新增校验**：
```javascript
// ✅ 方案三：补充 PUT 路由的校验（与 POST 保持一致）
// 验证 targetLevel（如果提供）
if (updates.targetLevel && !['ad', 'adset', 'campaign'].includes(updates.targetLevel)) {
  return res.status(400).json({ 
    error: 'targetLevel 必须是 ad、adset 或 campaign',
    code: 'INVALID_TARGET_LEVEL'
  })
}

// 验证 logicOperator（如果提供）
if (updates.logicOperator && !['AND', 'OR'].includes(updates.logicOperator)) {
  return res.status(400).json({ 
    error: 'logicOperator 必须是 AND 或 OR',
    code: 'INVALID_LOGIC_OPERATOR'
  })
}

// 验证 targetIds（如果提供，必须是数组）
if (updates.targetIds && !Array.isArray(updates.targetIds)) {
  return res.status(400).json({ 
    error: 'targetIds 必须是数组',
    code: 'INVALID_TARGET_IDS'
  })
}
```

**影响**：
- ✅ PUT 路由校验覆盖与 POST 保持一致
- ✅ 防止通过更新接口写入不合法值
- ✅ 更彻底的"长效防御"

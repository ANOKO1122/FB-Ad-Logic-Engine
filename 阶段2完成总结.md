### 步骤3：验证摘要写入

```sql

SELECT 
  run_id,
  rule_id,
  rule_name,
  account_id,
  status,
  skip_reason,
  matched_count,
  evaluated_at
FROM rule_execution_summaries
ORDER BY evaluated_at DESC
LIMIT 20;

-- 按 run_id 分组查看（同一轮执行的摘要）
SELECT 
  run_id,
  COUNT(*) AS summary_count,
  MIN(evaluated_at) AS start_time,
  MAX(evaluated_at) AS end_time
FROM rule_execution_summaries
GROUP BY run_id
ORDER BY start_time DESC
LIMIT 10;

-- 查看不同状态的分布
SELECT 
  status,
  COUNT(*) AS count
FROM rule_execution_summaries
GROUP BY status;

-- 查看不同跳过原因的分布
SELECT 
  skip_reason,
  COUNT(*) AS count
FROM rule_execution_summaries
WHERE skip_reason IS NOT NULL
GROUP BY skip_reason
ORDER BY count DESC;
```

## 🔍 排障提示

如果摘要没有写入：

1. **检查 SQL 是否执行**：确认表已创建
2. **检查规则是否执行**：查看控制台日志，确认规则执行流程是否走到
3. **检查错误日志**：查看是否有 `❌ 插入规则执行摘要失败` 的错误
4. **检查数据库连接**：确认 `pool.execute` 正常工作

如果查询接口返回 403：

1. **检查 token**：确认使用的是 admin 账号的 token
2. **检查用户角色**：确认 `req.user.role === 'admin'`

# 规则执行性能问题分析

## 🔍 问题根源

### 用户情况
- **1个规则**：指定了1个账户的41个广告
- **执行结果**：在**36个账户**上都执行了规则评估
- **性能问题**：26个账户超时（5分钟）

---

## ⚠️ 核心问题

### 当前执行逻辑

```javascript
// executeAllRules 会获取所有账户
const [rows] = await pool.execute(
  `SELECT DISTINCT fb_account_id FROM account_mappings WHERE is_active = 1`
)
targetAccountIds = rows.map(row => row.fb_account_id)  // 36个账户

// 为所有账户并发执行
await Promise.allSettled(
  targetAccountIds.map(accountId => executeRulesForAccount(accountId, { force }))
)
```

**问题**：
1. **获取所有账户**：即使规则指定了 `accountId`，仍然获取所有36个账户
2. **在所有账户上执行**：为所有36个账户并发执行规则评估
3. **内部过滤**：只有在 `executeRulesForAccount` 内部才检查规则是否匹配

### 执行流程

```
executeAllRules()
  ├─ 获取所有36个账户
  ├─ 为每个账户并发执行 executeRulesForAccount()
  │   ├─ 获取锁
  │   ├─ 查询数据库获取规则
  │   ├─ 检查规则是否匹配当前账户（第245行）
  │   │   └─ 如果不匹配，跳过（但已经查询了数据库）
  │   └─ 如果匹配，执行规则评估
  └─ 36个账户都执行完成
```

**问题**：
- 即使规则指定了 `accountId`，仍然会在所有账户上：
  1. 获取锁
  2. 查询数据库获取规则
  3. 检查规则是否匹配
  4. 如果不匹配，跳过（但已经浪费了时间）

---

## 🎯 解决方案

### 方案1：在 executeAllRules 中过滤账户（推荐）

**优化逻辑**：
- 如果规则指定了 `accountId`，只在该账户上执行
- 如果规则没有指定 `accountId`，在所有账户上执行

**实现**：

```javascript
export async function executeAllRules(options = {}) {
  const { force = false, accountIds = null } = options
  
  // 1. 获取所有启用的规则
  const enabledRules = await db
    .select()
    .from(rules)
    .where(eq(rules.enabled, true))
  
  // 2. 分析规则，确定需要执行的账户
  let targetAccountIds = accountIds
  if (!targetAccountIds) {
    // 获取所有规则指定的账户ID
    const ruleAccountIds = enabledRules
      .map(rule => rule.accountId)
      .filter(id => id != null)
    
    if (ruleAccountIds.length > 0) {
      // 如果规则指定了账户，只在这些账户上执行
      const [rows] = await pool.execute(
        `SELECT DISTINCT fb_account_id FROM account_mappings 
         WHERE is_active = 1 AND fb_account_id IN (?)`,
        [ruleAccountIds]
      )
      targetAccountIds = rows.map(row => row.fb_account_id)
    } else {
      // 如果规则没有指定账户，在所有账户上执行
      const [rows] = await pool.execute(
        `SELECT DISTINCT fb_account_id FROM account_mappings WHERE is_active = 1`
      )
      targetAccountIds = rows.map(row => row.fb_account_id)
    }
  }
  
  // 3. 只在目标账户上执行
  // ...
}
```

**优势**：
- ✅ 只在实际需要的账户上执行
- ✅ 减少不必要的锁获取和数据库查询
- ✅ 大幅提升性能

---

### 方案2：在 executeRulesForAccount 中提前返回（临时方案）

**优化逻辑**：
- 在获取锁之前，先检查规则是否匹配当前账户
- 如果不匹配，直接跳过，不获取锁

**问题**：
- 仍然需要查询所有规则，性能提升有限

## 📝 总结

### 问题根源

1. **执行范围过大**：即使规则指定了 `accountId`，仍然在所有账户上执行
2. **不必要的开销**：35个账户都获取锁、查询数据库，但最终都跳过
3. **性能浪费**：导致26个账户超时

### 解决方案

**推荐**：在 `executeAllRules` 中先分析规则，只在实际需要的账户上执行

**效果**：
- ✅ 只执行1个账户（而不是36个）
- ✅ 减少35个账户的不必要执行
- ✅ 大幅提升性能

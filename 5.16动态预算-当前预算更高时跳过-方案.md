# set_dynamic_budget 添加"当前预算更高时跳过"可选项

> 日期：2026-05-16 | 类型：功能增强 | 优先级：P2

---

## 一、需求

`set_dynamic_budget` 动作新增可选开关：

> **当公式计算结果低于当前预算时，不做下调**（如当前预算 $50，公式算出 $30，跳过不调）

复用现有"预算已达目标"的预飞行跳过链路，写入 `lastStatus: 'success'` 计入冷却。

## 三、代码修改

### 3.1 前端 RuleManager.vue

在 `set_dynamic_budget` 模板的预算上下限输入之后，`</template>` 之前，加一行复选框：

```html
<label class="skip-higher-wrap">
  <input type="checkbox" v-model="a.skip_when_current_higher" />
  <span>如果当前预算更高，不做下调</span>
</label>
```

同时在 `onActionTypeChange` 中初始化新字段默认值：

```js
} else if (a.type === 'set_dynamic_budget') {
  a.value = undefined
  a.value_unit = 'usd'
  a.metric = a.metric || 'purchases'
  if (a.multiplier == null || a.multiplier === '' || Number(a.multiplier) <= 0) a.multiplier = 30
  a.skip_when_current_higher = a.skip_when_current_higher ?? false  // 新增
}
```

提交 payload 时透传：

```js
if (a.type === 'set_dynamic_budget') {
  out.metric = a.metric || 'purchases'
  out.multiplier = Number(a.multiplier)
  out.value_unit = 'usd'
  delete out.value
  if (a.skip_when_current_higher) out.skip_when_current_higher = true  // 新增
  if (a.min_daily_budget != null) out.min_daily_budget = a.min_daily_budget
  if (a.max_daily_budget != null) out.max_daily_budget = a.max_daily_budget
}
```

### 3.2 前端 AdminTemplates.vue

同步 RuleManager.vue 的改动（复选框 UI + 初始化 + 提交）。

### 3.3 后端校验 templateValidator.js

`_validateActions` 中 `set_dynamic_budget` 分支，允许新字段（不需要额外校验，布尔值默认 `false` 即可）：

```js
// skip_when_current_higher 为可选布尔，不校验，仅透传
```

无需改校验逻辑，因为字段不存在时自然为 `undefined`/`false`。

### 3.4 执行层 actionExecutorService.js

在 `case 'set_dynamic_budget'` 中，`budget_already_at_target` 检查之后、执行 API 之前，插入：

```js
// 可选项：当前预算高于公式结果时，不做下调
if (action.skip_when_current_higher && targetContext.currentCents > dynamicBudget.finalBudgetCents) {
  status = 'skipped'
  errorMessage = '跳过下调：当前预算高于公式结果'
  apiRequest = JSON.stringify({
    preFlight: true,
    cooldownKey,
    currentCents: targetContext.currentCents,
    newBudgetCents: dynamicBudget.finalBudgetCents,
    reason: 'current_budget_higher',
    dynamicBudget
  })
  apiResponse = JSON.stringify({ skipped: true, reason: 'current_budget_higher' })
  break
}
```

**插入位置**：第 860 行 `budget_already_at_target` 检查的 `break` 之后，`logger.info(执行...)` 之前。

## 四、行为推演

| 场景 | 公式结果 | 当前预算 | 开关 | 结果 |
|------|---------|---------|:--:|------|
| 正常上调 | $80 | $50 | 任意 | 执行，预算调到 $80 ✅ |
| 正常下调 | $30 | $50 | 关 | 执行，预算调到 $30 |
| **跳过下调** | $30 | $50 | 开 | **跳过**，写 `success` 计入冷却 ✅ |
| 已达目标 | $50 | $50 | 任意 | 跳过（已有逻辑） |
| 公式算出 $0 | $0 | $50 | 开 | 不触发（$0 被 `computeDynamicBudgetCents` 拦截） |

---

## 五、冷却行为

跳过的状态写的是 `success`（与"预算已达目标"一致），计入 24h 冷却。

> 这些跳过虽然没调 API，但规则确实"评估并决策了"——决策结果是"当前预算更高，无需下调"。计入冷却避免每分钟重复评估同一个无需调整的广告。

## 二、Where（作用对象）重新设计：三层结构 + 两种模式
### 1）三层结构（保持你现有 targetLevel：Campaign / AdSet / Ad）
- Campaign：适合“系列级”控盘（例如节日活动整体止损）
- AdSet：适合“受众/版位/预算组”控盘（最稳定、运营最常用）
- Ad：适合“素材级”精细化（需要强识别信息，避免误选）
### 2）两种模式（关键）：手动选择 vs 智能筛选
在“选择目标对象”区域加一个切换（Tab 或 Segmented）：
 A. 手动选择（ID 模式）
适用：少量目标、明确知道要选哪几条 必须具备：

- 状态过滤（默认只看活的） ：默认 ACTIVE （可让用户切到 ACTIVE+PAUSED / ALL ）
- 列表身份证信息 （至少做到你截图痛点解决）：
  - 状态点（🟢ACTIVE / ⚪PAUSED / 🔴DISAPPROVED / ⚫ARCHIVED/DELETED）
  - 名称 + ID
  - 关键指标：Today Spend、ROAS、最近同步时间
  - 层级上下文：至少先显示 AdSet ID（后续补 AdSet/Campaign 名称） B. 智能筛选（Filter 模式，强烈建议作为“默认推荐”）
适用：你说的“十几个负责人用、靠命名批量管理” UI 结构建议拆成 3 组筛选器（运营能理解、也方便你实现）：

1. 状态 ：ACTIVE（默认）/ ACTIVE+PAUSED / ALL
2. 发布时间/新旧 ：近 N 天“发布/上新”的广告
3. 命名规则 ：名称包含/不包含（支持多个关键词）
并且必须有：

- 命中预览 ：实时展示“命中数量 + 前 50 条示例”
- 保存的是筛选条件，不是 ID ：执行时动态计算命中集合，新增同类广告自动纳入

## 四、When（触发条件）新增 4 个指标：单次加购/结账/添加支付/购买成本
你要加的其实是 4 个“漏斗单次成本”指标。建议你在条件 metric 下拉里新增：

- 单次加购成本（CPA_ATC / CP_ATC）
- 单次结账成本（CPA_IC / CP_IC）
- 单次添加支付信息成本（CPA_API / CP_API）
- 单次购买成本（CPA_PURCHASE）——你现在的 CPA 可以明确标注为“单次购买成本”

并且要以中文的形式在前端展现，因为运营不一定知道CPA_ATC这些是代表什么。
### 计算口径（离线一致、避免字段缺失）
统一规则： 优先用快照里已有成本字段；没有就用 spend / count 回退计算

- 例如 CP_ATC：
  - 若有 add_to_cart_cost 且 >0 → 用它
  - 否则若 add_to_cart_count >0 → spend / add_to_cart_count
  - 否则为 null（条件比较时视为不满足）
并且这些指标必须支持你现有 time_window（今天/昨天/近3天/累计）：

- today：从 ad_snapshots（热数据）直接取或计算
- yesterday/last_3_days/lifetime：从 daily_stats（冷数据）聚合（如果 daily_stats 没有某个 count，就从 actions JSON 解析补齐，或把这些 count 也落到 daily_stats 里）
## 五、广告名包含/不包含：放在 Where（而不是 When）
你提的“后续我们可以改广告名，通过广告名称来筛选哪些广告要创建哪些规则”，本质是 作用对象选择策略 ，不应当做触发条件（When），否则语义会混乱。

### 推荐的命名筛选 UI（运营可读、可维护）
在智能筛选（Filter 模式）里做成：

- 名称包含（任意一个关键词命中即可 OR / 必须全部命中 AND 可选）
- 名称不包含（黑名单词）
- 可选：正则（高级开关，默认隐藏，避免误用）
示例：

- 包含： US_Men_ 、 Video
- 不包含： test 、 old 、 archived
这样你们后续只要改名加前缀/标签，就能“规则自动覆盖”。

## 六、规则数据结构（保证你系统可演进、且兼容旧规则）
你现在规则里是 targetLevel + targetIds 。建议升级为：

- targetMode : "IDS" | "FILTER"
- targetIds : 旧的保留
- targetFilter : 新增
示例（FILTER 模式）：

```
{
  "targetMode": "FILTER",
  "targetLevel": "ad",
  "targetFilter": {
    "statusMode": "ACTIVE",
    "createdWithinDays": 7,
    "name": {
      "includeAny": ["US_Men_", "2026_"],
      "excludeAny": ["test", "archived"]
    }
  }
}
```
执行逻辑：

- IDS：按 id 精准集合
- FILTER：每次执行时先查询命中集合，再套 When 条件，再执行动作 这样你要的“新增同类广告自动生效”天然成立。

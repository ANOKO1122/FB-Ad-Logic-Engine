## 一、本对话窗口遇到的问题与解决方法

### 1. Vite / 测试报错：structureSyncService.js 第 316 行附近 "invalid JS syntax"

- **现象**：`npm test -- --run` 或前端构建时报错，指向 `structureSyncService.js` 第 316 行附近存在无效 JS 语法。
- **原因**：`listStructureObjectsFromDb` 的 JSDoc 注释块缺少开头的 `/**`，导致整段被解析为代码而非注释。
- **解决**：在该 JSDoc 块前补上 `/**`，修复后 24 个测试文件、182 个用例通过。

### 2. 用户验证时：SQL 如何核对「名称包含 0227 + 近 24 小时内创建」的三个广告

- **现象**：用户需要 SQL 命令在库中验证选中的三个广告 ID 是否同时满足「名称包含 0227」和「近 24 小时内创建」。
- **解决**：根据项目表结构（`structure_ads` 表、`ad_id` 列、`created_time` 存 ISO8601 字符串），给出两条 SQL：  
  (1) 仅校验名称 + 创建时间段（不限制 effective_status）；  
  (2) 与列表默认行为一致时增加 `AND effective_status = 'ACTIVE'`。  
  阈值使用 `CONCAT(DATE_FORMAT(DATE_SUB(UTC_TIMESTAMP(), INTERVAL 24 HOUR), '%Y-%m-%dT%H:%i:%s'), '.000Z')` 与后端 `scope_created_within_hours` 的 ISO 阈值口径一致。

### 3. 接口返回条数与 SQL COUNT 不一致（用户侧已理解）

- **现象**：同一条件下，SQL 直接 COUNT 与接口返回条数不一致。
- **原因**：接口在未传 `include_paused` 时默认只返回 ACTIVE（effective_status 过滤），SQL 若未加 `effective_status = 'ACTIVE'` 会包含 PAUSED 等。
- **解决**：在文档与说明中明确「与接口对齐的 SQL 需加上与当前请求一致的状态条件」，用户已按此理解并完成验证。

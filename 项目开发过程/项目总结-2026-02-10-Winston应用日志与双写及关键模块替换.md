- **范围**：`server/utils/logger.js`（新建）、`server/server.js`（logger + 异常/拒绝接管）、cronService / ingestorService / structureSyncService / rateLimitService / socks5 / index / db/connection / drizzle / actionExecutorService / ruleDataService / routes(rules,system,admin) / middleware(authJwt) 中 `console.*` 替换为 logger；`docs/Step4_体验与完整性_验收与下一步.md` 补充 Winston 验收步骤；`package.json` 增加 `dev:server:logs` 脚本。

## 二、对话内容概要

- 用户约定：**命令由用户手动执行**，**代码修改由 AI 完成**，开发遵守 `.cursorrules`。
- 先完成：新建 `server/utils/logger.js`，`server.js` 引入 logger 并接管 `uncaughtException` / `unhandledRejection`，Step4 文档补充 Winston 验收步骤。
- 用户验证：`/api/health` 通过、开发模式终端有 Winston 格式输出；生产模式下一度看不到 `logs/` 文件夹。
- 排查后：生产模式仅配置了文件 transport，未配置 Console，导致终端无输出；用户澄清需求为「终端照常输出 + 同时写文件保留 14 天」。
- 最终：logger 改为**开发/生产均带 Console**，生产额外增加文件 transport（双写）；关键模块全部改为 logger，保证业务日志也进文件。

## 三、遇到的问题与解决方法

| 问题 | 原因/现象 | 解决方法 |
|------|------------|----------|
| **1. 生产模式看不到 logs 文件夹** | 仅配置了文件 transport，且依赖首次写盘才建目录，部分环境未及时创建。 | 在 logger 加载时对生产模式执行 `fs.mkdirSync(logDir, { recursive: true })`，并打印 `[Winston] 日志目录: <绝对路径>`，便于确认位置。 |
| **2. 生产模式终端没有任何日志** | 生产只配置了文件 transport，未配置 Console。 | 改为「开发仅 Console，生产 = Console + 文件」双写，终端与文件同时输出。 |
| **3. 用户需求理解偏差** | 初版理解为「生产只写文件、终端安静」。 | 用户明确：终端要有实时输出，同时文件里保留 14 天便于事后排查；按「双写」调整 logger 配置。 |
| **4. 业务日志仍打终端、不进文件** | 仅 server.js 用了 logger，cron/ingestor/socks5/index 等仍用 console。 | 在 cronService、ingestorService、structureSyncService、rateLimitService、socks5、index、db/connection、drizzle、actionExecutorService、ruleDataService、routes(rules,system,admin)、authJwt 中引入 logger，将 `console.log/warn/error` 全部替换为 `logger.info/warn/error`。 |
| **5. 启动方式记忆成本** | 生产双写需执行 `set NODE_ENV=production&& node server/server.js`。 | 在 package.json 增加脚本 `dev:server:logs`，内部执行上述命令（Windows）；文档说明 Mac/Linux 可用 `NODE_ENV=production node server/server.js`。 |

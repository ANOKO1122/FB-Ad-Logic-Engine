# 云端项目总结 · 2026-04-11 · 规则审计页体验增强与双栏摘要

## 1. 背景与目标

在 M4.4 **人为规则审计基线**（列表/详情 API、`snapshot_before` 字段 diff、管理端页）之上，按子计划 [规则审计日志详细执行方案_7ffb510d.plan.md](../.cursor/plans/规则审计日志详细执行方案_7ffb510d.plan.md) 提升**可读性与筛选体验**，且遵守 **P0：不改动** `GET /api/admin/rule-history` 与 `GET /api/admin/rule-history/:id` 的响应形状；所有叙述与双栏展示均为**读时计算**。

主方案台账已同步：[终版云本地协同方案_6100b6d5.plan.md](../.cursor/plans/终版云本地协同方案_6100b6d5.plan.md) 中 `rule-audit-ux-enhancement` todo 为 **completed**，§4.4 与修订记录已更新。

## 2. 本次开发窗口：问题与处理

| 现象或矛盾 | 原因与处理 |
|------------|------------|
| 长句「由…改为…」让 IF、复杂字段难读 | 详情主区改为**双栏**（修改前 / 修改后），各列用人话格式化；复杂结构仍保留折叠 **JSON** 对照。 |
| 曾尝试双栏只显示「增减」导致丢失全量上下文 | 产品要求**全量快照**可审计；已恢复双栏为**全量**，再在集合类字段**上方**叠加「本次增减摘要」（新增 / 减少分开展示）。 |
| `exclude_ids`、`target_by_account` 结构复杂 | 摘要按层级或分桶：`exclude_ids` 按广告/广告组/广告系列分层；`target_by_account` 区分新桶、移除桶、同账户下 ID 增减（与子计划 §10.4 一致）。 |
| 旧数据或解析路径异常导致白屏 | `ruleAuditNarrative.js` 内 **safeArray / safeExcludeIds**、单字段 **try-catch**；`AdminRuleHistory.vue` 中 **detailViewOk** 与底部**原始详情 JSON** 兜底（子计划 §10.5）。 |
| 字段展示顺序混乱 | 统一 **`DIFF_FIELD_RANK`** + `sortRuleHistoryChanges`，禁止纯字母序。 |

## 4. 建议验证步骤

1. **构建与单测**：在项目根目录执行 `npm run build`、`npm run test`，应通过（与开发过程中一致）。
2. **浏览器**：以 `admin` / `super_admin` 登录 → 打开 **规则审计** 页（`/admin/rule-history`）。
3. **筛选**：选择日期范围后点击「查询」，确认列表仍按原 API 行为拉取。
4. **详情**：打开一条含字段 diff 的记录，确认：
   - 字段顺序符合运营习惯（非纯字母序）；
   - 集合类字段：**上方增减摘要** + **下方双栏全量**；
   - 解析异常时出现降级句且仍可展开原始 JSON。

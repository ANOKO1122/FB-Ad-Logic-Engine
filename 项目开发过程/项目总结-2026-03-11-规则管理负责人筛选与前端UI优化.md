## 二、遇到的问题与解决方法

### 2.1 规则测试 targetIds 断言失败（expected [] to deeply equal ['adset_123', ...]）

**现象**：`npm test -- --run server/tests/rules.test.js` 中 3 个用例失败——创建/获取/更新规则时期望 `targetIds` 为 `['adset_123', 'adset_456']` 等，实际返回 `[]`。

**根因**：服务端 `syncTargetByAccount` 只接受**复合 ID** 格式（`accountId:objId`），`parseCompositeId('adset_123')` 无冒号返回 null，该项被过滤掉，导致 targetIds 被归一化为空数组。测试用例仍按「纯 ID」传参与断言，与实现约定不一致。

**解决**：采用方案 A——修改测试以符合当前实现。在 `server/tests/rules.test.js` 中，创建/更新时 `targetIds` 改为 `[${TEST_ACCOUNT_ID}:adset_123`, `${TEST_ACCOUNT_ID}:adset_456]` 等复合 ID，断言同步改为期望复合 ID 数组。修改后 18 个用例全部通过。

### 2.2 负责人筛选触发器点击无反应、控制台报 "not defined on instance"

**现象**：管理员进入规则页后，负责人筛选触发器显示「负责人: ▼」，点击无反应；控制台报 `Property "ownerFilterOpen" was accessed during render but is not defined on instance` 及 `ownerFilterDisplayValue` 同理。

**根因**：RuleManager.vue 使用 **Options API** 的 `setup()` 且通过 `return { ... }` 暴露给模板。负责人筛选用到的 `ownerFilterOpen`、`ownerFilterDisplayValue`、`ownerDropdownRef`、`clearOwnerFilter`、`toggleOwnerOption` 在 setup 内已定义，但**未加入 return 对象**，模板访问时实例上不存在。

**解决**：在 `setup()` 的 `return` 中补充上述 5 项，使模板能正确绑定与调用。修改后下拉展开/收起、清空、多选与列表刷新均符合预期。

## 三、已完成的实现与验收

### 3.1 后端（方案 B + 联表）

| 层级 | 文件 | 改动要点 |
|------|------|----------|
| Schema | `server/db/schema.js` | 新增 `users`、`owners` 最小只读表定义（id、owner_id→ownerId；id、owner_name→ownerName），供联表 |
| Service | `server/services/rulesService.js` | `getUserRules` 改为 from(rules).leftJoin(users).leftJoin(owners)；select 显式扁平化（getTableColumns(rules) + ownerId/ownerName）；仅当 isAdmin 且 ownerIds 非空时 inArray(users.ownerId, ownerIds) |
| Route | `server/routes/rules.js` | GET /api/rules 解析 req.query.ownerIds 为数字数组并传入 options.ownerIds；不传或空表示不按负责人过滤 |

### 3.2 前端

| 位置 | 改动要点 |
|------|----------|
| 规则列表 | 仅管理员显示「负责人」自定义多选下拉：触发器展示「负责人: 全部 (不选=全部)」或已选负责人逗号拼接；清空按钮、展开/收起箭头；下拉内「清除筛选 (全选)」+ 负责人列表（自定义复选框、选中蓝底白勾）；点击外部关闭 |
| 规则卡片 | 仅管理员显示「负责人: xxx」或「未分配」 |
| loadRules | 当 isAdmin 且 selectedOwnerIds.length > 0 时请求带 `?ownerIds=1,2,3`；首屏根据 data.isAdmin 拉取 GET /api/owners 填充下拉选项 |
| normalizeRule | 映射 ownerId、ownerName 供卡片展示 |

### 3.3 测试与文档

- **规则 API 测试**：targetIds 改为复合 ID 格式，18 个用例全部通过。
- **验证文档**：`docs/规则管理负责人筛选_验证与测试.md`（自动化测试说明、手工验证步骤、接口直测、常见问题）。

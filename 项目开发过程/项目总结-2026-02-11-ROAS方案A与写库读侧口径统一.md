# 项目总结：2026-02-11 ROAS 方案 A 与写库读侧口径统一

本窗口完成 ROAS「0 vs null」语义统一（方案 A）：写侧不持久化派生 cpc/ucpc/roas，只存分子/分母与 API 兜底 roas；读侧 ruleDataService 统一「计算优先→API兜底→零兜底」；聚合使用总分子/总分母；parseActions 保守防双算、extractPurchaseValue 白名单扩展；Warn 限流；历史 daily_stats 清洗完成。下文为对话内容概要、修改点、改前改后、验收、下一窗口建议。

---

## 一、本窗口目标与范围

- **目标**：统一 ROAS/CPC 等比值类指标的写库与读库口径，避免「spend>0 且 value=0 时 roas=0」与「不可计算时 roas=null」混用导致的规则误判。
- **范围**：ingestorService（extractRoas/extractApiRoas/extractPurchaseValue/parseActions、saveSnapshotsToDbInternal、updateDailyStatsFromInsights、archiveDailyStats、fields 清单、Warn 限流）；ruleDataService（calculateSingleDayMetrics、queryAdSnapshots/queryDailyStats 映射、aggregateMultiDayMetrics）；ruleDataService.test.js、ingestorService.test.js 用例；历史 daily_stats 清洗 SQL。

---

## 二、对话内容概要

- 用户约定：遵守 `.cursorrules`，命令由用户手动执行。
- 核心诉求：ROAS 在 spend>0 且 value=0 时应为 0（可比较），不可计算时应为 null（不触发比较）；写库不存派生值，只存分子/分母与 API 兜底。
- 实现路径：**方案 A**——写侧三条路径统一不写 cpc/ucpc，roas 仅存 API 兜底或 null；读侧「计算优先→API兜底→零兜底」；聚合用总分子/总分母，避免辛普森悖论。
- 用户修复：登录接口路径 `/auth/login` → `/api/auth/login`；curl 示例修正为正确 URL 与单行命令。

## 三、修改点与改前改后

### 3.1 写库路径（ingestorService）

| 路径 | 改前 | 改后 |
|------|------|------|
| **saveSnapshotsToDbInternal** | cpc/ucpc/roas 写入 API 或计算值 | cpc/ucpc=null；roas 仅存 extractApiRoas(insight) 或 null |
| **updateDailyStatsFromInsights** | cpc/roas 写入 API 或计算值 | cpc=null；roas=extractApiRoas(insight) 或 null |
| **archiveDailyStats** | cpc/roas 写入计算或 API 值 | cpc=null；roas 仅 API 兜底或 null |

### 3.2 读库路径（ruleDataService）

| 逻辑 | 改前 | 改后 |
|------|------|------|
| **calculateSingleDayMetrics** | 混合使用计算/API/兜底 0 | 统一「计算优先→API兜底→零兜底」：spend==0→null；value>0→value/spend；day.roas>0→day.roas；否则 0.0 |
| **aggregateMultiDayMetrics** | 部分使用 AVG(roas) | 使用 SUM(purchase_value)/NULLIF(SUM(spend),0) 总分子/总分母 |
| **queryAdSnapshots / queryDailyStats 映射** | 直接透传 roas | 统一由 calculateSingleDayMetrics 或 aggregate 计算，不信任持久化 roas |

### 3.3 提取逻辑（ingestorService）

| 函数 | 改前 | 改后 |
|------|------|------|
| **extractApiRoas** | 无 | 新增：从 purchase_roas / website_purchase_roas 取首个有效值 |
| **extractPurchaseValue** | 白名单有限 | 扩展 action_type 白名单（omni_purchase、website_purchase 等），加 trim 与 Number.isFinite 校验 |
| **parseActions** | 可能双算 | 保守防双算：omni_purchase 优先；否则取各渠道 max；purchase 兜底 |

### 3.4 其他

| 项 | 改前 | 改后 |
|------|------|------|
| **自动任务 fields** | 含 cpc | 去掉 cpc，增加 website_purchase_roas |
| **Warn 限流** | 可能多条 | purchases>0 且 value=0 且 apiRoas=null 时，按 syncSessionId 输出 1 条汇总 warn |
| **历史 daily_stats** | cpc/roas 有值 | 清洗：UPDATE ... SET cpc=NULL, roas=NULL |

## 七、重要口径备忘

- **daily_stats.roas / ad_snapshots.roas**：语义为 ROAS_Fallback_Only，仅作 API 兜底，读侧不信任。

- **Warn 日志**：`[ROAS] sync_session_id=xxx purchases>0 且 purchase_value=0 且 api_roas=null：count=N, sample_ad_ids=...`

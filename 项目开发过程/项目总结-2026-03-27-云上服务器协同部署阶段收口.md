## 三、问题与解决方法（云上实战复盘）

### 问题 1：`/ronghe/` 返回 200 但实际是 FB 页面

- 现象：访问 `/ronghe/` 命中 FB 的 `location /` 回退。
- 原因：未显式配置 `/ronghe/` 代理规则。
- 解决：在 FB Nginx 增加 `/ronghe` 与 `/ronghe/` 专用 location。

### 问题 2：`/ronghe/` 页面资源路径错误导致空白/MIME 问题

- 现象：控制台出现脚本 MIME/资源路径异常，或页面空白。
- 原因：融合平台构建仍按根路径 `/` 出资源。
- 解决：
  - `vite.config.js` 设置 `base: '/ronghe/'`
  - 重新 `npm run build`
  - Nginx 补充 `/ronghe/` 子路径兼容。

### 问题 3：视频生成接口在 `/ronghe/` 下 404

- 现象：`/api/video-generation/api/check-ffmpeg`、`/api/image-links` 返回 404（HTML）。
- 原因：请求被错误路由到 FB 后端。
- 解决：在 FB Nginx 增加融合平台 API 前缀固定分流到 `ronghe-nginx`。

### 问题 4：重建后短时 502

- 现象：刚重建/重载后接口偶发 502。
- 原因：上游容器短暂启动窗口（upstream 还未 ready）。
- 解决：等待服务 ready 后复测，接口恢复 200 JSON。

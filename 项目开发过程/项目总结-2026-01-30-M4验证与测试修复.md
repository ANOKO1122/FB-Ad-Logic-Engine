- **结论**：M4 的 3.2～3.5 核心逻辑均通过 Vitest 单元测试验证。

### 2. 规则 API 测试修复（12 个失败 → 通过）

- **原因**：M3 后创建规则必填 `accountId`，且该账户须属于当前用户的 `owner_id`（在 `account_mappings` 中存在且 is_active=1）；旧测试未传 accountId，测试用户也无 owner/account_mapping。
- **修复**：
  - beforeAll：创建测试用 owner → 创建用户并绑定 owner_id → 插入 account_mappings（fb_account_id=act_test_rules）。
  - 所有 POST /api/rules 请求增加 `accountId: 'act_test_rules'`。
  - 「应该拒绝缺少必填字段的请求」改为传 accountId、只缺 conditions/actions，以测到 MISSING_FIELDS。
  - afterAll：按顺序清理规则、account_mappings、用户、owner。

### 3. 时间窗口测试修复（1 个失败 → 通过）

- **原因**：`last_3_days` 实现为「2 天前到今天」共 3 天（start = now.minus({ days: 2 })），测试误用「3 天前」作期望，导致 start.day 与 expectedStart.day 不一致（如 31 vs 30）。
- **修复**：将期望改为 `expectedStart = now.minus({ days: 2 }).startOf('day')`，与实现一致。

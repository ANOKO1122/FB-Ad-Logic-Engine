## 二、本窗口遇到的问题与解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| **排除区搜名称能搜到、搜广告 ID 搜不到** | 后端 `structureSyncService.js` 的 listStructure* 对 `q` 仅加 `name LIKE ?`，无按 id 过滤；目标区在 RuleManager 对「纯数字且长度≥10」做了 isIdSearch + resolveObjectsByIds 合并，排除区没有 | 在 `refreshExcludeScopeItems` 中：当关键词满足 `/^\d{10,}$/` 且**单账户**时，与列表请求并行调用 `resolveObjectsByIds(keyword)`，将解析结果中未在列表中且非 missing 的项合并进排除区列表，并补 `account_id: ids[0]`；多账户与目标区一致不做 ID 解析合并（后端 resolve 不返回 account_id） |
| **规则 743/744 无执行日志** | 用户误以为应产生执行记录 | 用 SQL 查 rule_matched_objects、ad_snapshots(data_date=今日)、rule_execution_summaries、account_mappings；结论：两规则匹配的广告在「今日 ad_snapshots」中 0 条（项目约定 today 只写 spend>0），规则引擎用 spend>0 时匹配数为 0，Cron 在无到期匹配时直接 return，不写 rule_execution_summaries，也不更新 last_executed_at；属预期，问题二视为正常 |

## 三、已完成的实现与验收

### 3.1 方案一：开启动态后展示「当前匹配数」

- **文件**：`src/views/RuleManager.vue`
- **实现**：新增 3 秒防抖 watch，依赖 `scopeConditionRows`、`selectedAccountIds`、`targetLevel`、`excludeTargetIds`、`maxDynamicMatches`、`useDynamicScope`；仅在开启动态且已选账户且存在无错误条件行时调用 `previewDynamicScope`；**只写** `ruleForm.matchedCount`，**不写** `targetIds`；使用 `buildScopeFiltersFromRows`、`buildExcludeIdsForPreview`。
- **验收**：开启动态、选账户、填完整监控条件后，防抖约 3 秒内「当前匹配 N 个对象」更新为预览接口返回的 count；做法 A 约定：targetIds 不被预览结果改写。

### 3.2 方案二：目标区搜索 = q + limit 50 + 不用 after

- **常量**：`SCOPE_SEARCH_LIMIT = 50`。
- **refreshScopeItems**：有关键词时 `limit=50`、`after=null`，请求成功后置 `scopePagingAfter = null`（搜索模式不显示「加载更多」）；无关键词仍为 limit=500 并保留 after。
- **验收**：输入关键词防抖后列表为服务端单页最多 50 条，无加载更多；清空关键词后恢复首屏 500 条及加载更多。

### 3.3 方案二：排除区独立搜索与列表

- **状态**：`excludeScopeItems`、`excludeScopeLoading`、`excludeRequestId`。
- **refreshExcludeScopeItems**：仅在 `excludeScopeSearch` 非空时请求，`q` + `limit=50` + `after=null`，结果写入 `excludeScopeItems`。
- **filteredExcludeScopeItems**：有关键词用 `excludeScopeItems`，无关键词用 `scopeItems`。
- **防抖 watch**：对 `excludeScopeSearch` 300ms 防抖触发 `refreshExcludeScopeItems`；无关键词时清空 `excludeScopeItems`。
- **加载状态**：`excludeAreaLoading`（有关键词用 `excludeScopeLoading`，无关键词用 `scopeLoading`）。
- **验收**：排除区输入关键词防抖后为独立请求的单页结果（最多 50 条）；清空后展示与目标区相同的 scopeItems；排除区不显示加载更多。

### 3.4 排除区按广告 ID 搜索（补充）

- **单账户**：当关键词满足 `/^\d{10,}$/` 时，与列表请求并行调用 `resolveObjectsByIds(keyword)`，将解析结果中未在列表中且非 missing 的项插入列表前部，并补 `account_id: ids[0]`。
- **多账户**：不做 ID 解析合并（与目标区一致；后端 resolve 不返回 account_id）。
- **验收**：单账户下在排除区输入广告 ID（如 120239512858390499）可搜到并可勾选排除。

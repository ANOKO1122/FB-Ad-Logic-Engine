## 一、本窗口完成的工作

### 1. 顺序1：数据口径验证闭环

- **现状核对**：确认后端 spend>0 过滤（3 处：API 嗅探、滑动窗口 Today、队列写入器）与 effective_status 写入 status 的位置（ingestorService.js 对应行号）。
- **验证方案**：编写 `docs/顺序1_数据口径_验证方案.md`，给出 SQL 1/2/3 与日志检查点；明确「队列写入器可能宽松、验收以 DB 为准」的细节风险点。
- **用户自验**：SQL 1（today 无 spend=0）通过、SQL 2（status 分布）正常、日志有过滤前后数量 → **顺序1 数据口径验收通过**。
- **结论**：today 只写 spend>0、ad_snapshots.status 来自 effective_status 已落地；口径验收以数据库结果为准，不因某段代码是否「严格 spend>0」而困惑。

### 2. 顺序2：前端指定对象选择器口径与任务

- **方案文档**：编写 `docs/顺序2_前端指定对象选择器_方案.md`（目标、步骤、风险、验证；当时为「默认 ACTIVE/PAUSED + 可切 ALL」）。
- **口径升级**：按「Read Local, Sync Remote」将选择器彻底改为查库、FB 只在「同步结构」时调用；写入 TASKS.md 与 DEV_PLAN.md：
  - **TASKS**：1.2 末尾追加 4 条（结构镜像表、元数据同步、新对象补全、手动刷新）+ 验收口径「选择器不穿透 FB」；2.3 最前插入 3 条（选择器查库、resolve 查库、刷新按钮）+ 顺序2 实施约束（after 游标、resolve 多类型、effective_status 口径、首次全量可观测）；补强权限隔离验收、resolve DB miss 产品行为、手动刷新接口位置、分页参数统一为 after。
  - **DEV_PLAN**：M3 Where 口径改为只查结构镜像表；4.1 增加 structure_* 表（字段、索引、同步策略、职责边界）；5.3 增加选择器接口口径。
- **实施约束**：顺序2 实施约束/自检（after 游标语义、resolve 同 ID 多类型、effective_status vs status、首次全量回填与可观测）写入 TASKS，落地时必守。

### 3. 比值类指标语义与「坏 vs 空」产品口径

- **问题**：link_clicks=0 时 Facebook 不返回有效 cpc，库里 cpc 为 null，规则「cpc > 0.8」永不触发，导致「有花费无点击」的广告不会被关。
- **口径融入**：
  - **DEV_PLAN**：新增 5.2.1 指标口径与规则引擎比较语义（cpc/ucpc/cpa/roas 分母为 0 时为 null；null 与数值比较恒为 false；坏 vs 空：贵点击、空耗、空转化）；M3 增加对比值类口径的引用；5.3 SOP 模板补充三条（空耗、贵点击、空转化）。
  - **TASKS**：AND/OR 验收示例（spend>1.5 AND link_clicks=0、link_clicks>0 AND cpc>0.8）；3 个规则模板任务（空耗/贵点击/空转化）；前端口径提示/防呆任务；比值类指标+模板 实现对齐自检（字段名对齐、比较器语义对齐、模板可验收性）。
- **立即可用**：运营可将规则改为「空耗关停」spend > 1.5 AND link_clicks = 0 或「贵点击关停」link_clicks > 0 AND cpc > 0.8，无需等开发。

## 二、本窗口遇到的问题与解决方法

| 问题 | 解决方法 |
|------|----------|
| 顺序1 验收时「队列写入器」可能不是严格 spend>0，存在宽松条件 | 在验证方案中写明细节风险点，强调「口径验收以 DB 为准」；SQL 1 通过即认定 today 只写 spend>0。 |
| 顺序2 要明确「选择器查库、FB 只在同步结构时调用」 | 将 Read Local/Sync Remote 写进 TASKS 与 DEV_PLAN：结构镜像表、默认隐藏 DELETED/ARCHIVED、q 非空放宽、resolve 不穿透、权限隔离、实施约束（after/resolve/effective_status/首次全量）。 |
| 用户通过他人 account_id 可能查到不属于自己的结构数据 | 在 TASKS「选择器改为查库」验收中增加：非 admin 用他人 account_id 调接口仍返回 403（或空），与当前权限隔离测试一致。 |
| 结构镜像表与 ad_snapshots 职责易混用 | 在 DEV_PLAN structure_* 段落增加：结构镜像用于选择器与回显；ad_snapshots/daily_stats 只用于指标计算与规则评估，二者不要混用。 |
| resolve DB miss 时前端产品行为不明确 | 在 TASKS resolve 验收中增加：DB miss 时 UI 仍能显示该 id（带 missing 提示），且不会阻断保存/编辑。 |
| link_clicks=0 时 cpc 为 null 导致 cpc 规则不触发 | 定死比值类指标语义（分母为 0 为 null）、规则比较 null 恒为 false；用空耗/贵点击/空转化模板拆分「坏 vs 空」；前端防呆提示 + 实现对齐自检（字段名、比较器、模板可验收性）。 |

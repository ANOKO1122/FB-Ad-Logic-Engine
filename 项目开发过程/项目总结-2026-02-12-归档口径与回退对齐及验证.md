## 一、本窗口目标与范围

- **目标**：统一每日数据按「事件发生的自然日（date_start）」入库；提升归档主/备路径稳健性；在冷数据层为 cpc/roas 提供合理单日兜底；便于长期观测回退使用情况。
- **范围**：ingestorService（updateDailyStatsFromInsights、archiveDailyStats、queryLastSnapshotCompatible、archiveAllAccountsDailyStats）、迁移 017、手动归档脚本、验证文档、单测（archiveFallback、archiveAllAccountsDailyStats.metrics）。

## 二、遇到的问题与解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| **初步归档失败：purchaseValue is not defined** | archiveDailyStats 中构造 daily_stats 插入值时使用了未定义的 `purchaseValue`，而数据来自 `row.purchase_value`。 | 在 values 的 map 内增加 `const purchaseValue = parseFloat(row.purchase_value ?? 0)`，插入时使用该变量。 |
| **3 个账户 ad_snapshots 无新同步（last_sync 停留在 1 月底/2 月初）** | 需区分是「同步成功但 0 条写入」还是「未参与同步」。原日志只有「有数据更新」的账户才打日志，无数据更新的账户无观测。 | 在统一心跳每账户同步完成后：① 当 synced 但无数据更新时打 `📋 [accountId] 同步完成但无数据更新（…）`；② 统一打 `📊 [accountId] 拉取 spend>0=N, 写入 ad_snapshots=N`（或滑动窗口的 todayCount/dailyStatsCount）。便于 grep 排查。 |
| **手动归档时 36 个账户全部「已归档且完整，跳过」，归档 0 条** | archiveAllAccountsDailyStats 在 forceAll=true 时仍做完整性检查，已完整则直接 return，不调用 archiveDailyStats，导致无法回填 cpc/roas。 | 仅在 `!forceAll && isComplete` 时 return 跳过；`forceAll` 时一律执行归档（并打「强制重跑归档」日志），便于手动回填兜底值。 |
| **回退路径（ROW_NUMBER 不可用时）使用 synced_at 范围，与主路径 data_date 口径不一致** | queryLastSnapshotCompatible 原为纯 synced_at 范围取数，主路径为 data_date，主备语义不一致。 | 回退函数改为：入参增加 targetDateStr；第一步按 data_date=targetDateStr 取最后快照；若结果为空则第二步按 synced_at 范围兜底并打 warn（含 accountId、dateStr、ads 数）。返回 { rows, mode, ads } 供上游打观测。 |
| **回退分支未定义 startTime/endTime 导致崩溃** | 回退分支调用 queryLastSnapshotCompatible(accountId, startTime, endTime) 但 startTime/endTime 未在 archiveDailyStats 中定义。 | 在计算 targetDateStart/End 后增加 `const startTime = targetDateStart.toJSDate()`、`const endTime = targetDateEnd.toJSDate()`，回退调用改为传入 dateStr 与 startTime/endTime。 |
| **迁移 017 执行时报 ERROR 1271 Illegal mix of collations for operation 'concat'** | 存储过程内 SELECT CONCAT('✅ …', …) 使用 emoji 等字符与参数 collation 混用导致 1271。 | 将提示文案改为纯 ASCII（如 "OK index added:" / "OK index exists, skipped:"），避免字符集冲突；索引仍成功创建。 |

## 三、本窗口修改点汇总

### 3.1 代码（server/services/ingestorService.js）

- **日期口径**：`updateDailyStatsFromInsights` 写入 daily_stats 的日期字段由 `insight.date || insight.date_start` 改为 `insight.date_start || insight.date`。
- **回退时间边界**：`archiveDailyStats` 在计算 targetDateStart/End 后定义 `startTime`/`endTime`，回退调用改为 `queryLastSnapshotCompatible(accountId, dateStr, startTime, endTime)`。
- **冷数据 cpc/roas 兜底（仅归档路径）**：archiveDailyStats 的 values 映射中：cpc 在 link_clicks>0 且 spend>0 时为 spend/link_clicks，否则 null；roas 优先 API 昨日值，否则在 purchase_value>0 且 spend>0 时为 purchase_value/spend，否则 null。
- **强制重跑**：`archiveAllAccountsDailyStats` 中仅在 `!forceAll && isComplete` 时跳过；forceAll 时打「强制重跑归档」并执行归档。
- **回退函数**：`queryLastSnapshotCompatible(accountId, targetDateStr, startTime, endTime)` 实现为 data_date 优先、synced_at 兜底；返回 `{ rows, mode: 'data_date'|'synced_at', ads }`；兜底时 warn 含 ads 数。
- **archiveDailyStats 返回值**：增加 `compat: { mode, ads }`（仅走兼容方案时有值）；回退分支赋值 compatMeta。
- **归档汇总观测**：archiveAllAccountsDailyStats 汇总日志增加「回退观测」：兼容方案账户数、synced_at 兜底账户数、synced_at 兜底广告数。
- **按账户观测**：统一心跳每账户同步完成后，无数据更新时打 📋、统一打 📊（拉取/写入 ad_snapshots 或 todayCount/dailyStatsCount）。

### 3.2 脚本与迁移

- **server/scripts/manual-archive.js**：新增，调用 manualArchive()，用于手动强制归档所有账户「昨日」。
- **server/db/migrations/017_add_idx_account_date_ad_idempotent.sql**：幂等添加索引 `idx_account_date_ad(account_id, data_date, ad_id)`；存储过程内 CONCAT 改为纯 ASCII 避免 1271。

### 3.3 单测

- **server/tests/archiveFallback.test.js**：① data_date 优先命中时 compat.mode='data_date'、去重后 ads=1、不触发 synced_at warn；② data_date 为空时走 synced_at 兜底、compat.mode='synced_at'、打 warn 含 ads。
- **server/tests/archiveAllAccountsDailyStats.metrics.test.js**：归档汇总日志包含「回退观测」及兼容方案账户数、synced_at 兜底账户数/广告数。

### 3.4 文档

- **docs/archive_date_start_and_cpc_roas_验证.md**：已有日期一致性、回退健壮性、cpc/roas 兜底、唯一索引、两表键一致性与「仅归档来源」CPC 验证、手动触发与验证步骤；本窗口未改结构，仅实际执行验证通过。

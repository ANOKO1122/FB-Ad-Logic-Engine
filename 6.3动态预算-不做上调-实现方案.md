# 动态预算：「如果当前预算更低，不做上调」实现方案

## 一、需求背景

在规则编辑 → 执行动作 → 设置动态预算值中，已有选项：

> ☑ 如果当前预算更高，不做下调

现新增对称选项：

> ☑ 如果当前预算更低，不做上调

**场景**：当公式结果 > 当前预算（本意是上调），但发现当前预算已被人工提前调得比公式结果更低时，若规则执行上调就会覆盖人工决策。勾选此选项后，规则将跳过本次预算调整。

**与已有选项的关系**：一次执行中公式结果与当前预算的大小关系是唯一的，两个选项互斥生效，不会同时触发。约定用户不会同时勾选两者。

> 由于 Dry Run 模式下没有真实的 `targetContext.currentCents`，需确认是否要在 Dry Run 中也模拟。建议：Dry Run 保持简单，不做跳过模拟（与 `skip_when_current_higher` 当前行为一致——当前 Dry Run 也没有检查 `skip_when_current_higher`）。

#### 4.2 真实执行模式（~第 868 行）

在 `skip_when_current_higher` 判断块之后、实际 API 调用之前，新增：

```javascript
// 可选项：当前预算低于公式结果时，不做上调
if (action.skip_when_current_lower && targetContext.currentCents < dynamicBudget.finalBudgetCents) {
  status = 'skipped'
  errorMessage = '跳过上调：当前预算低于公式结果（已被人为降低）'
  apiRequest = JSON.stringify({
    preFlight: true,
    cooldownKey,
    currentCents: targetContext.currentCents,
    newBudgetCents: dynamicBudget.finalBudgetCents,
    reason: 'current_budget_lower',
    dynamicBudget
  })
  apiResponse = JSON.stringify({ skipped: true, reason: 'current_budget_lower' })
  break
}
```

#### 4.3 审计日志写入

无需额外改动——`status='skipped'` + `errorMessage` 会正常写入 `automation_logs` 表。

## 四、执行流程（完整）

```
公式计算 → finalBudgetCents
    ↓
解析预算目标 → resolveBudgetTargetContext() 调 FB API 获取 currentCents
    ↓
currentCents === finalBudgetCents  → 跳过（budget_already_at_target）
    ↓
currentCents > finalBudgetCents
    → skip_when_current_higher? → 跳过（current_budget_higher）
    → 否则执行下调
    ↓
currentCents < finalBudgetCents
    → skip_when_current_lower?  → 跳过（current_budget_lower）  ← 新增
    → 否则执行上调
```

## 六、验收要点

1. 规则管理中新建/编辑动态预算动作，能看到新增的 checkbox
2. 勾选「不做上调」后保存，刷新页面回显示仍为勾选状态
3. 卡片视图摘要中显示「当前预算更低时跳过」
4. 模板管理中同样支持该选项
5. 审计日志 `automation_logs` 中，被拦截的记录 `status='skipped'`，`error_message='跳过上调：当前预算低于公式结果（已被人为降低）'`
6. 不勾选时行为不变，正常执行预算调整
7. `skip_when_current_higher` 原有功能不受影响
